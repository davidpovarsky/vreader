## Project-Specific Instructions

Custom instructions appended to Codex's developer-instructions for every command.

```text
This is a Swift 6 iOS ebook reader app using strict concurrency (SWIFT_STRICT_CONCURRENCY=complete).

Key conventions:
- All SwiftData mutations go through PersistenceActor. Never return @Model instances across actor boundaries — use value-type DTOs (BookRecord, HighlightRecord, etc.).
- ViewModels and observable stores are @MainActor @Observable final class.
- Cross-actor calls use await. No assumeIsolated except in narrow App.init contexts.
- No bare print() in production — use Logger(subsystem: "com.vreader.app", category: "...").
- All JS string interpolation into WKWebView evaluateJavaScript must go through FoliateJSEscaper.
- Reader bridges attach UIKit observers/gesture recognizers in attach(to:) and must release them in deinit.
- Code files should stay under ~300 lines. Split proactively.
- Swift Testing is the default test framework (import Testing, @Test, #expect). XCTest only for XCTestExpectation / notification timing tests.
- DEBUG-only code must be wrapped in #if DEBUG at file scope, never inline in production code.
- FormatCapabilities controls which reader features (TTS, highlights, paging, unified reflow) are available per format.
```
