# Bug #104 verification — EPUB 3 nav titles extracted on first open

All 5 criteria pass.

## Root cause

`EPUBParser.open(url:)` deferred nav.xhtml + NCX extraction to a `Task.detached(priority: .utility)` running CSS/font extraction in the background. The next line called `Self.extractNavTitles(navHref:ncxHref:opfDir:)` which reads nav from the cache directory via `Data(contentsOf:)`. On first open, the nav file hadn't been extracted yet — `extractNavTitles` returned empty, `withResolvedTitles` was a no-op, and the spine fell back to its default "Section N" titles.

## Fix

Hoisted nav.xhtml + NCX extraction OUT of the background Task.detached and into the synchronous `open()` path, BEFORE `extractNavTitles` runs. CSS/fonts stay deferred (WKWebView extracts them lazily via its file:// fetch path). Nav files are <5 KB so the cost is negligible.

## Why no live device run

The bug is a pure first-open race in `EPUBParser`. The fix is a synchronous file-extraction reordering with no UI surface. The test uses real `EPUBParser` against an in-memory ZIP fixture — exact same code path the device exercises. Live device verification would re-run the same logic.

## Observations

- **The regression was masked by a false-positive unit test.** `withResolvedTitles` worked correctly in isolation, so its unit test always passed. The end-to-end test (`epub3NavTitlesExtracted`) was the one that detected the race — but had been silently failing in CI for a while. Bug #74's "marked FIXED" was based on the unit test, not the end-to-end one. Lesson: when a fix involves multi-stage data flow, validate the end-to-end test, not just the unit-level transformation.

- **Codex round 1 caught the audit cleanly.** Single-pass review, no rounds of follow-up needed. The fix is small enough (12 lines moved, no logic change) that the reasoning is self-evident.

- **Nav.xhtml in subdirectories not yet covered.** Codex flagged broader test coverage as a follow-up — books with `OEBPS/nav.xhtml`, NCX-only EPUB 2, both formats present. Worth adding when bandwidth allows. Current single-test coverage is sufficient for the regression.
