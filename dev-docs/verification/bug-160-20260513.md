## Observations

- The UITest uses `--seed-position-test` (Position Test Book, ~5000-char TXT, no chapter markers = single-chapter/continuous mode). This exercises the non-chapter path that was wired in the first partial fix.
- Chapter-mode path (the WI-3 fix — `makeLocatorForTXT` global offset translation) is covered by unit tests (`TXTChapterHighlightCreationTests`). A runtime chapter-mode UITest would require a fixture with detectable `CHAPTER I/II/III` markers; `war-and-peace.txt` qualifies but the UITest fixture is the more reliable single-chapter seed.
- Chrome toggle: after the Highlight context menu item is tapped, `isChromeVisible` remains `true` (the long-press does NOT post `readerContentTapped`). Earlier test iterations that added an unconditional `textViewForChrome.tap()` were toggling chrome off (hiding the annotations button). Fixed by checking `annotationsButton.waitForExistence(timeout: 2)` before deciding whether to tap.
- The `app.menuItems["Highlight"]` approach (iOS 16+ `UIEditMenuInteraction` path) worked immediately without needing the `app.buttons["Highlight"]` fallback.
