## Observations

- **The fix's mechanism** (per the bug row): replacing the
  `@Observable`-incompatible `didSet`-with-self-write pattern by
  moving the clamp into the setter directly (or writing to a
  `_storage` backing without going through the synthesized public
  setter). Whatever the chosen fix path, the runtime outcome is that
  setting `interval = anyValue` no longer re-enters its own setter
  via the `@Observable` macro's synthesized accessor.
- **Why this verify is conclusive**: the pre-fix stack-guard fault
  triggered within ~23.7k frames — that's milliseconds at modern
  iOS clock rates. If the recursion had survived the fix, the app
  would have died inside `AutoPageTurnerSettings`'s reader-side init
  long before the library view could compose. The fact that the app
  not only launches but renders the reader content (which requires
  AutoPageTurnerSettings to have been read AND applied) is direct
  evidence that the setter completes cleanly.
- **No regression observed** in adjacent surfaces: settings still
  read from defaults (the `3` echoed back); reader rendered with the
  multi-page fixture (no obvious rendering glitches); chrome bar +
  scrubber rendered correctly.
