## Observations

- **The Round-1 "code-read root cause" was disproven.** The bug doc's
  2026-05-18 runtime-instrumented iteration was correct: `pageNavigator`
  is NOT nil; paged mode DOES engage. The visible symptoms come from (a)
  chrome occlusion of the rendered content + bottom-of-VStack page
  indicator, (b) the paginator's `NSTextContainer` being sized to
  `UIScreen.main.bounds.size` rather than the renderer's actual smaller
  box, (c) no tap recognizer on `NativePagedContainer` at all (the SwiftUI
  layer attached zero gesture handlers).
- **Codex audit Round-1 caught a load-bearing nuance I missed.** Threading
  `proxy.size` into the paginator isn't enough — the renderer's effective
  text box is `proxy.size − chrome-aware bottom padding −
  indicator-when-hidden − 2 × textContainerInset`. Round-2 added the
  `paginatorViewportSize` helper + the `lineFragmentPadding = 0` parity
  setting. The fix only works because both adjustments land together.
- **Chrome toggle is its own re-paginate trigger.** A `proxy.size` change
  isn't sufficient because `proxy.size` reports the GeometryReader's slot
  size, not the inner VStack's content area — toggling `isChromeVisible`
  doesn't change the proxy size, just the padding inside. The
  `.onChange(of: isChromeVisible)` handler is the second trigger.
- **DebugBridge re-grant required after every reinstall.** The URL-scheme
  grant is keyed to the installed app's UUID — `xcrun simctl install`
  produces a new UUID, so `scripts/grant-debug-scheme-approval.sh` must
  run between install and the first `openurl` call. Re-discovered as a
  papercut here; documented for future verification runs.
- **The booted simulator UDID was `61149F0E-…`, not the brief's
  `1FAB9493-…`.** Followed the project's
  `feedback_xcodebuild_sim_udid.md` rule to pin to the booted-sim's UDID
  rather than chase the brief's value (which was a stale leftover from a
  prior session).
