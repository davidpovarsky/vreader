# Bug #239 — Paged-layout side-tap page-turn producer restoration

## Repro context

Bug filed 2026-05-19: in Paged layout, side-tap page-turn does nothing in
the TXT, EPUB, and AZW3/MOBI readers (and MD, PDF). Root cause investigated
by a read-only subagent: feature #54 WI-3 (commit `e30f769`) deleted
`ReaderUnifiedDispatch.swift`, the only mount of `.tapZoneOverlay(config:)`.
`TapZoneOverlay` / `TapZoneDispatcher` was the sole producer of
`.readerNextPage` / `.readerPreviousPage`; once it was unmounted, every
native reader container's `onReceive` observer went dead.

This verification confirms the producer is restored without re-mounting
the deleted SwiftUI overlay (which had swallowed scroll gestures on
UIKit-backed renderers per bug #70).

## Out of scope

- TXT lacks a paged renderer entirely (`TXTReaderContainerView` body has no `NativeTextPagedView` branch). The producer fix restores the notification stream; the missing renderer is a separate piece of work (see bug #239 row's investigation note). Tracking will be filed as its own bug when picked up.
- EPUB cross-chapter advance on right-zone tap from the last page of chapter N. Pre-existing behavior: the EPUB `.readerNextPage` observer calls `pageNavigator.nextPage()` (within-chapter), not a chapter-bump. Out of scope for #239 (which is a producer regression, not a navigator-scope enhancement). The reader-navigation design (§2.2) calls for hybrid wrap-on-boundary as future work — distinct row when picked up.
- Notification scoping by `fingerprintKey` / `readerToken` is pre-existing architectural debt: the `.readerNextPage` / `.readerPreviousPage` contract has been unscoped since feature #25's original landing (the deleted `TapZoneDispatcher.dispatch` also posted `object: nil`). Codex audit flagged this as Medium with a multi-scene `WindowGroup` failure mode; vreader is single-window today (no `UIApplicationSupportsMultipleScenes` in `Info.plist`), so the failure mode is hypothetical. Documented in the audit log; filed as follow-up below.

## Follow-up

- **Notification scoping by reader identity** — file a follow-up bug to scope `.readerNextPage` / `.readerPreviousPage` / `.readerContentTapped` by `fingerprintKey` + `readerToken` (matching the DebugReaderRegistry precedent from #142). PDF additionally lacks a paged-layout guard on its `.readerNextPage` observer (gated implicitly by the router today; defense-in-depth). This is a feature-shaped refactor on the producer/consumer contract, not a bug fix on #239.
- **TXT paged renderer** — file a follow-up bug for the deeper TXT-paged layer: `TXTReaderContainerView` body has no `NativeTextPagedView` branch and `updatePaginationIfNeeded()` is defined but never called, so TXT "Paged" is single-chapter scroll regardless of the producer state.
- **EPUB cross-chapter advance on side-tap** — the reader-navigation design (`reader-navigation.md` §2.2) calls for a "hybrid" model where right-zone tap from the last page of chapter N wraps to chapter N+1. Today the EPUB `.readerNextPage` observer is within-chapter only. Tracked as future design-driven work, not part of #239.
