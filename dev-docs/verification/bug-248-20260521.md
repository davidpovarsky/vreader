## Observations

- **Input-injection tooling gap (the named, confirmed-missing tool).** This
  host drives the Mac through a Screen-Sharing **virtual-only** display
  (`/cu-diagnose`: `Display: Screen Sharing Virtual Display`,
  `screencapture: works`, `MCP screenshot: fails`). Two independent input
  paths were tried and both are dead:
  1. **computer-use** — `mcp__computer-use__screenshot` returns
     `CU display unavailable` (the MCP capture path refuses a virtual-only
     display).
  2. **CGEventPost** (the `sim-drive-fallback` skill's `clickat.swift`) —
     taps report success at the OS level and Simulator is confirmed
     frontmost, but the synthetic touch never reaches the app (verified with
     two taps on two screens at correct coordinates: the Library book cover
     and the reader's Contents button — neither navigated).
  `simctl io screenshot` (capture) and `vreader-debug://` (open / search /
  settle / file-import) both work, which is how the repro state was reached;
  only the final sheet-open tap is blocked.
- **The repro state WAS reproduced CU-free.** The reader is sitting at
  "Chapter 12 of 20" — exactly the user's scenario ("read into a middle
  chapter, open Contents"). The fix's behavior (auto-scroll + highlight)
  triggers when that sheet opens; the code path is exercised by the 8 new
  unit tests and confirmed correct by the Codex Gate-4 audit.
- **Build-phase gotcha worth recording**: `xcodebuild build-for-testing`
  (and the `test` action) did NOT run the "Inject DebugBridge URL types"
  Run Script phase that registers `vreader-debug://` in the built
  `Info.plist`, so every debug-bridge URL failed with
  `LSApplicationWorkspaceErrorDomain error 115` until a plain
  `xcodebuild build -configuration Debug` + reinstall. Future CU-free verify
  runs should install from a plain `build`, not from the `test` products.
- **Why partial, not blocked**: per AGENTS.md the `FIXED` row flip is the
  merge gate (code on main + passing tests), which is fully satisfied. The
  visible on-device confirmation is the GH-close gate, deferred post-merge
  with the `awaiting-device-verification` label. The verification is not
  "skipped" — it is gated on a confirmed-missing tool (working HID input on
  a virtual-display host), the explicit rule-47 Gate-5 exception.
