## Observations

- **The detent param works exactly as designed.** Screenshot A (`.medium`) and B
  (`.large`) differ unambiguously: A is a half-height sheet showing the language
  rail + the top of the tall ORIGINAL; B is a full-height sheet exposing far more
  of the `translationResultCard`. The `@State aiPanelDetent` â†’ `presentationDetents(selection:)`
  binding animates the already-open sheet to `.large` when the command re-fires â€”
  no dismiss/re-present needed.
- **`translationResultCard` is one ScrollView with two sibling cards** (original +
  Spanish translation), not a card stacked below a separate ORIGINAL. So the
  "ORIGINAL" text visible in B IS the top of `translationResultCard`. With the
  war-and-peace fixture's long Chapter 1, the Spanish sub-card sits lower in that
  inner ScrollView; `.large` maximizes the visible window (the PR's job), and the
  remaining reach is an in-card scroll. For a shorter passage the Spanish card
  would fit at `.large` outright.
- **AI gating gotcha confirmed** (matches the Feature #69 learning): `--enable-ai`
  alone is NOT enough to fire a real translate â€” `AIService.sendRequest` guards on
  `featureFlags.aiAssistant` + `consentManager.hasConsent` independently. Both
  UserDefaults keys had to be set (`com.vreader.featureFlags.aiAssistant`,
  `com.vreader.ai.consentGranted`) + cfprefsd bounced before launch.
- **Brittle for next time**: `seedWarAndPeace` hashes the actual fixture file, so
  the fingerprintKey is `txt:bd8285a8â€¦:1705`, NOT the `â€¦beef01` literal that
  appears in the seeder source (that literal is dead/overwritten). Read the real
  key from the seeded `ImportedBooks/txt_<sha>_<bytes>.txt` filename rather than
  trusting the source literal.
- The Gate-4 round-1 reset (`translationViewModel.reset()` on a fresh
  `.translate` present) is correct production parity, but it means a verify must
  fire the translate AFTER the `detent=large` present, not re-present after a
  completed translate (the re-present clears the prior result). Sequenced
  accordingly.
