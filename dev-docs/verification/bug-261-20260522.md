# Bug #261 — AZW3/MOBI font-size overshoot (em-compounding) — fix verification

CU-free verification via the `vreader-debug://` DebugBridge `eval` of
`getComputedStyle` inside the Foliate content document
(`document.getElementById('view').renderer.getContents()[0].doc`). Run on a
dedicated, uncontended iPhone 17 Pro Max Simulator (`6C32EE30…`) — both
iPhone 17 Pro sims were occupied by concurrent agents (`LSApplicationWorkspace`
error 115 risk). For a font-size calibration verification (eval-based, not
safe-area layout), 17 Pro Max is equivalent to 17 Pro (same precedent as the
feature #70 Gate-5 evidence).

## Root cause diagnosed (candidate 2: em-compounding, NOT multiplier mis-tune)

Both EPUB and AZW3 use `foliate: 1.12` / `epub: 1.12`. At unified 28 the
calibrated body px is identical (~31px), so candidate 1 (multiplier overshoot)
is **ruled out**. The overshoot came from the book's own `em`/`%`-based
descendant CSS compounding against the injected `body` base, because
`FoliateStyleMapper` pinned ONLY `body` font-size and lacked the
cascade-flatten rule EPUB already has (`ReaderThemeV2+EPUBCSS.swift`).

## Known residual boundary (accepted, shared with EPUB — Gate-4 Medium)

A book using `!important` on its OWN font-size (`p{font-size:1.15em !important}`)
or a higher-specificity selector can still win at equal `!important` precedence
+ later source order — measured `<p>` stayed 35.65px under that artificial
stress. This is the **same boundary EPUB has** (its `inherit !important` reset
loses to a book's `!important` rule injected later too) and is NOT a feature-#70
regression. Real Kindle/KFX content CSS overwhelmingly uses plain `em`/`%`
WITHOUT `!important` (criterion 2's realistic case, fully flattened). The fix
widens the Foliate reset list beyond EPUB's (adds `section, article, aside,
main, header, footer, figure`) for extra immunity. Over-flattening to a
universal `* { font-size: inherit }` was rejected: it would diverge AZW3 from
EPUB in the opposite direction and break legitimate book typography EPUB
preserves via `revert`.
