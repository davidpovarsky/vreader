# Bug #268 — Bilingual unit-model divergence (translate-block-text-directly + Foliate leaf-fix)

PR that references this file.

## Why the exception path

The end-to-end verification (render a nested-block EPUB → run AI translation →
observe a complete 1:1 interlinear render) requires the AI translation path, which
is NOT CU-free-drivable (no `--enable-ai` + provider-config DebugBridge harness for
the bilingual path — same class as #267/#243), AND a nested-block EPUB fixture
(verification debt). The FIX LOGIC, however, is fully exercisable at the real
subsystem boundaries by deterministic unit tests, so per AGENTS.md the
high-fidelity tests satisfy the close gate.

## Sub-parts

- **(1) EPUB translate-block-text-directly** (this PR): when the plain-text
  prefetch's segment count diverges from the DOM leaf-enumerate's block count
  (nested `<pre>`-blank-lines / mixed-content `<blockquote>`), the inject path
  translates the enumerate's OWN block `text[]` directly (`translatePreSegmented`
  → `translatedSegmentsDirect` → `translateBlocksDirectly`), making blocks↔segments
  1:1 by construction. Additive + low-risk: the common matched-count path is
  untouched; the fallback only activates on the rare mismatch (currently source-only),
  so it can only improve or no-change — never regress or produce a wrong pairing.
- **(2) Foliate-host leaf-fix** (shipped v3.39.32, PR #1183): the AZW3/MOBI
  `bilingualEnumerate` skips non-leaf blocks, mirroring the EPUB #266 fix.
