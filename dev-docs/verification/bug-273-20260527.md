# Bug #273 device verification — DebugBridge `navigate` driver + feature #71 WI-8 continuous-mode navigation

The fix adds `vreader-debug://navigate?spine=<N>[&fraction=<F>]` (drives
`.readerNavigateToLocator` CU-free) and a `multi-chapter-epub` fixture (4
viewport-tall chapters). This verification confirms BOTH the new harness works
AND that feature #71 WI-8's continuous-mode navigation (in-window scroll AND
out-of-window rebuild) functions on device.

Note: `commit_sha` above is the `main` baseline at verification time; the fix
was exercised from the `fix/issue-1203-debugbridge-navigate-driver` branch
build (it merges with this evidence). The build was installed via
`simctl install` (data-preserving) resolved from `BUILT_PRODUCTS_DIR`.

## Observations

- The `mini-epub3` fixture (the only EPUB fixture before this) is too short to
  verify scroll navigation: its 2 tiny chapters total ~947px and fit in roughly
  one viewport (`maxScroll=53`), so any `navigate` clamps to the 53px bottom and
  can't be told apart. This is precisely why Bug #273 needed BOTH a driver and a
  taller fixture — the driver alone (verified working) couldn't be measured.
- In-window navigation lands at exact `offsetTop`; out-of-window lands at
  `offsetTop − 77px` (the safe-area inset). Both show the target chapter at the
  top, but the 77px landing-consistency difference between the two branches is a
  polish nit for feature #71's terminal WI (documented in the `docs/features.md`
  #71 row), not a navigation failure.
