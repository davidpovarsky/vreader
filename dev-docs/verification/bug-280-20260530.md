## Observations

The 1.12 was never the "small lift for sub-pixel rendering" the comment
claimed — it was a flat 12% over-inflation for EPUB. The `-apple-system` CSS
stack resolves to the SAME SF Pro face UIKit uses, and WebKit maps 1 CSS px to
1 UIKit point at the pinned `initial-scale=1`/`maximum-scale=1` viewport (bug
#279), so EPUB was already at parity before any multiplier — 1.12 actively
broke the parity the calibration system exists to deliver.

Foliate is genuinely a bit smaller-capped (15.89 vs 16.91) because its default
UA font (no body `font-family` is injected) is NOT the same face as EPUB's
`-apple-system` stack — so the two web targets correctly get DIFFERENT
multipliers, which the old shared-1.12 assumption never captured.

Brittle-for-next-time: the measurement reads Canvas `actualBoundingBoxAscent`
(ink height of 'H') against UIKit `UIFont.capHeight` (a font metric). These
agreed to <0.03 px here and the size-invariance control confirms linearity,
but a future OS/font change could shift the numbers — re-run
`FontSizeCalibrationMeasurementTests` to re-derive rather than hand-editing the
literals.

This evidence's `commit_sha` is the base (b709c97e) the measurement ran
against; the fix lands on `fix/bug-280-font-calibration` with the re-tuned
literals + the measurement test. The measured ratios are deterministic and
reproduced identically across three runs.
