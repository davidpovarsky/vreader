# Bug #314 — AI Translate translates book-context, not the user's selection (reopened)

**Verification-exception**: the failure is a SwiftUI view-timing race — a touch text
SELECTION made while the AI sheet is ALREADY open, where `onChange(of: showAIPanel)`
doesn't fire so the parked selection was never applied → cold context (front-matter).
Reproducing it CU-free needs a synthesized WKWebView long-press selection while a sheet
is presented; the available tooling can't (CU display flapped to unavailable; idb not on
PATH). High-fidelity deterministic tests drive the same decision + translate path.

## Observations

- Root cause was precise: `onChange(of: showAIPanel)` is a no-op when the panel is already
  open, so the parked-selection apply (which lived only there) was skipped for a
  select-while-open. The fix factors a shared `applyPendingTranslateSelection()` called from
  BOTH the handler's already-open branch AND onChange, with a pure `isExplicitSelection`
  decision.
- The select-while-sheet-open touch interaction is the un-reproducible part (CU flapped, no
  idb); the decision + verbatim translate are the deterministically-tested code the fix routes through.
