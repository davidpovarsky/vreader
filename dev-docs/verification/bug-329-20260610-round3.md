# Bug #329 round 3 — re-close verification at 1px granularity (GH #1568)

The reopen's re-close bar: **a 1px-granularity DebugBridge scroll sweep across
≥2 chapter boundaries on an image-bearing EPUB shows monotonic scrollTop/
progress — no backward jumps, no stalls — including a pass while images are
still decoding. A 60px sweep does NOT satisfy re-close.** The goal additionally
required ≥5 chapters on the-half-second.epub.

## Acceptance criteria

| Criterion | Observed | Result |
|---|---|---|
| 1px granularity | Each tick: `scrollTop += 1` + scroll dispatch, in-page recorder samples every tick (`scripts/b329-1px-sweep.sh`) | ✅ |
| ≥5 chapter boundaries, the-half-second.epub | **11 boundaries**: spine 0→7 (sweep 1) + 7→11 (sweep 2) on "The Half Second" (54-spine real EPUB) | ✅ |
| Image-bearing + decode window | Sweep 1 starts seconds after a FRESH app launch + open (cover/typewriter images decoding); each mid-sweep extend materializes sections whose images decode during the sweep; the long chapters (7→11) are the image-bearing region that exposed the residual | ✅ |
| Monotonic content position (no backward jumps) | Front matter: **0** jumps / 3512 ticks. Long chapters: **0** jumps / 11541 ticks | ✅ |
| No stalls | 0 (both sweeps reached their target spine, reason=target) | ✅ |
| No window thrash | A→B→A count 0; window transitions all monotonic (6 + 5) | ✅ |

## Baseline (pre-fix main, same harness)

25 backward jumps, 7 window-collapse cycles ([2,3,4,5]→[0,1,2]), reader trapped
at spine 2 after 4557 ticks, terminating at a fake document end
(`artifacts/bug-329-r3-baseline-1px-20260610.json`).

## The two shipped fixes

1. **v3.62.5 (`0894bf6f`, PR #1624) — px-aware eviction deferral**: the
   geometric root cause; defer any trim that would leave the trailing side
   below `prefetchPx(800)+128`; observer reports pxAbove/pxBelow/
   sectionHeights; ResizeObserver compensates above-viewport async growth.
2. **v3.62.6 (`b7652503`, PR #1625) — disconnected-section guard**: the
   formal sweep itself caught 5 residual jumps in the long chapters (scrollTop
   crashing 11308→278 at multi-evict moments): an evicted section's terminal
   zero-size ResizeObserver entry double-subtracted on top of the removal's own
   compensation (`artifacts/bug-329-r3-residual-5jumps-20260610.json`).

## Observations

- The fixed window dynamics are textbook: spans float to 5–6 sections through
  the short front matter (deferred eviction), evictions land only once ≥864px
  of content remains on the trimmed side, and every eviction's compensation
  leaves scrollTop above the prefetch threshold — no echo, no oscillation.
- The residual found mid-verification (and fixed before close) underlines the
  reopen's premise: only the 1px sweep exposes this class — both prior rounds
  passed 60px sweeps.
