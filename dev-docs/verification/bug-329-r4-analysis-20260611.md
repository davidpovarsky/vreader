# Bug #329 round 4 — root-cause analysis (2026-06-11)

## Repro (REAL gestures, user-class book — the round-4 bar)

- Book: 道诡异仙 (19.4MB CJK novel, 1042 spines, image chapter headers) —
  `epub:1f5aecb8…:19381838`, imported from `test-books/books/epub/`.
- Engine: legacy EPUB continuous scroll (Scroll layout), iPhone 17 Pro sim.
- Driver: `scripts/b329-gesture-probe.sh` — idb HID pans (duration 1.1–1.2s;
  sub-1.0s pans don't register; pans must start below the chrome strip and
  with chrome dismissed). A PASSIVE in-page `scroll`-event recorder
  (`window.__g329`) logs `(n, scrollTop, vsi, intraPx)` per event — no
  synthetic stepping.
- 20 forward pans + 12 forward/forward/backward reversal cycles → 4449
  scroll events: `dev-docs/verification/artifacts/bug-329-r4-passive-recording-20260611.json`.

## Findings

- Between-pan samples (60-pan sweep): **0 backward jumps** — the round-3
  geometric guard holds for the settled states the 1px harness measures.
- WITHIN gestures: **77 scrollTop teleports** (|Δ|>200px between consecutive
  scroll events) in compensation/re-assert pairs, e.g. `10352→6276→10362`
  (−4076 / +4086), `10711→2603→6539→10641` (−8108 double-evict / re-assert).
- **43 content-position leaps** paired with them: ev606–650 shows the reader
  racing spine 80 → 99 (~19 chapters in ~1s of touch) — each cycle advances
  one whole section. This is the user's "scrolling isn't smooth; it often
  jumps".

## Mechanism (the round-3 verification gap)

A JS `scrollTop -= removedHeight` eviction compensation issued while a pan
gesture is ACTIVE is overridden on the next frame by the scroller's internal
gesture anchor (UIScrollView re-asserts the touch-tracked contentOffset).
Net effect: the DOM above the viewport shrank by `removedHeight` but
scrollTop snaps back → the content visually leaps FORWARD by the removed
height → the boundary observer immediately fires again → extend+evict →
another leap. The synthetic 1px DebugBridge ticks set scrollTop with NO
active gesture, so the compensation always stuck — rounds 1–3 never saw it.
The ResizeObserver growth compensation has the same exposure during touch.

## Fix direction (round 4)

Gesture-aware mutation deferral:
1. Observer JS tracks `touchActive` (touchstart/end/cancel, passive) and
   reports it in every boundary signal; on touchend it emits a settle signal.
2. Coordinator: while `touchActive`, forward APPENDS stay allowed (no
   compensation needed) but EVICTIONS and backward PREPENDS (anything
   mutating content above the viewport, i.e. anything that compensates
   scrollTop) are DEFERRED to the first post-touch signal.
3. ResizeObserver compensation: queue deltas while `touchActive`; apply once
   on touchend.
4. `StitchSimulator` upgrade: model the gesture-anchor override (a JS
   scrollTop write during `touchActive` is restored to the gesture position
   on the next event). CONTROL case must reproduce the runaway
   (ev606-style); the deferral case must be monotonic.

## Re-close bar additions

Re-run this probe post-fix: 0 content leaps + 0 teleport pairs during pans;
between-pan monotonic; the 1px synthetic sweep stays clean (no regression of
rounds 1–3); reversal cycles clean.
