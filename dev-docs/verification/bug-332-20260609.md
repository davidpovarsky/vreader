## Observations

- The legacy continuous-scroll engine (default scroll layout → `epubWKWebView` stitch)
  is where the bug lived; the rewriter's per-chapter absolutization had a gap for
  `xlink:href`. Default Readium renders scroll per-spine (no stitch) so it was unaffected.
- The Codex audit caught that SVG2 `href` (no xlink:) is the same bug class for modern
  EPUBs — fixed with an element-scoped pass before this verification.
- `renderedW/H = 370×493` (non-zero) is the load-success signal — a 404'd broken glyph
  renders at ~0×0.
