# Bug #356 — canonical identity diverges/collapses on edge-cases (NFC + non-finite)

Two defects: (1) no NFC normalization of canonical string fields → NFD-vs-NFC
text hashed differently within iOS and across platforms; (2) non-finite
`progression` silently omitted → an invalid locator canonicalized identically to
a valid missing-progression one, colliding on the derived key. iOS portion
delivered via feature #109 (WI-1 NFC + recompute backfill, WI-2 non-finite
persistence guarding).

## Verification class: exception (high-fidelity integration test)

The non-finite-locator failure mode is a **fault-injection path**: a non-finite
`progression` does not arise from normal device reading (the validated
`LocatorFactory` rejects it; the pre-fix `?? Locator(...)` fallbacks were the
only way it leaked, and only under a degenerate division). It cannot be
reproduced through the device UI without a fault-injection harness. Per AGENTS.md
close-gate, verified via a deterministic high-fidelity integration test that
drives the **real subsystem boundary** (the actual `PersistenceActor` +
`ModelContainer`, not stubs), plus an on-device launch confirming the NFC
recompute backfill runs over real persisted data.
