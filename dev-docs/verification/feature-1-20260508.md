## Summary

Round-2 slice for feature #1 (Edit and delete bookmarks — High).
Extends the 2026-05-07 evidence (`feature-1-20260507.md`) which
verified the data layer via 14 unit tests on `BookmarkListViewModel`,
`PersistenceActor` bookmark CRUD, and `BookmarkFeedback` haptic with
`result: partial — UI gesture for swipe + rename deferred`.

This round closes the **annotations panel UI** leg via two XCUITest
suites totalling 11 methods (188.1 s wall-clock):

- `vreaderUITests/Annotations/AnnotationsPanelPlaceholderTests` (7
  methods) — verifies the panel opens, has 4 tabs (Bookmarks /
  Contents / Highlights / Notes), each tab shows its placeholder
  empty state, and the panel passes the iOS 26 accessibility audit.
- `vreaderUITests/Reader/ReaderAnnotationsPanelTests` (4 methods) —
  verifies the panel presents from the reader chrome's Annotations
  toolbar button, tabs are present, tab-switching works (synthesized
  taps), and the panel dismisses cleanly.

Together these close the panel-presence + panel-navigation legs that
sit upstream of every per-tab interaction (bookmark rename, swipe
delete, highlight edit, note edit). The per-tab interaction gestures
themselves are still deferred — they require a populated reader with
seeded annotations, which the placeholder tests don't drive.

All 11 synthesized real taps via XCUIApplication, no Computer Use,
no display capture. Target simulator was iPhone 17 (iOS 26.4
Simulator). Same harness pattern as PRs #419 (#6 round-2) and #420
(#9 round-2).

## Observations

- During this verification iteration, an unrelated XCUITest suite
  (`PositionPersistenceTests`) was discovered to be regressed —
  3/3 tests fail because the `.positionTest` seed path doesn't make
  the seeded book reachable through tap-to-navigate. Filed as bug
  #150 (GH #421). Production tap-to-reader path is fine; the
  regression is narrowly in test infrastructure / seeder.
- The annotations panel placeholder tests run quickly because they
  hit empty state — no seeded bookmarks/highlights/notes. The
  per-tab interaction gestures (rename, swipe-delete, edit) need a
  follow-up suite that seeds annotations first. That's the
  remaining gap before #1 can flip to `VERIFIED`.
