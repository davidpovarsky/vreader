# Feature #101 — In-reader total reading time — Gate 5b acceptance

Plan: `dev-docs/plans/20260611-feature-101-reading-time.md`. Design:
`vreader-reading-time.jsx` (`RTMetricsLine` / `RTBottomChrome` /
`RTBookDetailsRows`), bundle PR #1652, design-notes §#1641. All three
WIs merged: WI-1 PR #1667 (v3.64.4), WI-2a PR #1668 (v3.64.5), WI-2b
PR #1669 (v3.65.0, merge `4e5ad40f`). Verification ran against the
merged `main` build installed over the existing simulator data
(real-library state: 7 books, mixed engines).

## Observations

- The Foliate initial-position settle ticks `updateTimeDisplays` during
  load, so the time readout is available almost immediately on AZW3 —
  the "inert tap" window is much shorter there than on Readium.
- The Half Second crossed 10h during verification (9h 12m → 10h
  observed across the two runs), conveniently exercising both total
  forms on a real book.
- Unit suites: 75 feature-#101 tests green across WI-1/2a/2b (formatter
  boundaries, readout seam, per-book persistence round-trip, per-book
  stats + first-date fetches, model derivations, row composition,
  mirror keying, stale-fetch races at both the lifecycle helper and the
  details fetcher).
