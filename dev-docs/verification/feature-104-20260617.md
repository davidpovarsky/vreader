# Feature #104 — Android Spike A: canonical cross-platform identity — verdict

The Risk-1 gate: prove/decide deterministic book identity + reading position
across Swift (iOS) and Kotlin (Android) BEFORE promising library/backup interop.
The deliverable is the **canonical-identity DECISION** (`contracts/identity/DECISION.md`),
backed by a running dual-platform conformance lane.

## Conformance results (the gate)

**Swift side** — `scripts/run-tests.sh vreaderTests/IdentityConformanceTests` →
`RUN-TESTS RESULT: SUCCEEDED` (3 tests: fingerprint, **locator**, cache-key).
**Kotlin side** — `contracts/conformance/kotlin ./gradlew test` → `BUILD
SUCCESSFUL` (same 3 vector sets). Both green against one shared `contracts/vectors/`
set ⇒ the cross-platform identity contract holds:

- **Fingerprint** (`{format}:{sha256}:{byteCount}`) — identical Swift↔Kotlin,
  round-trip + invalid-rejection parity.
- **Cache key** (`book|unit|lang|prompt`, provider excluded) — identical.
- **Locator canonical serialization** (`Locator.canonicalJSON`) — a reading
  position serializes **byte-identically** Swift↔Kotlin (sorted keys, `%.6f`
  POSIX floats, normalized line-endings, RFC-8259 escaping) → identical
  `canonicalHash` → cross-platform position identity for dedup/sync.

## The decision (one line)

Exact-match for native fingerprint + cache-key + engine-neutral
`Locator.canonicalJSON`; **source-bytes** (not converted-EPUB) for
converted-Kindle cross-platform identity; **CFI lossy-fallback** (cross-device
EPUB resume = progression + text-quote anchors). No global normalization layer.
Full rationale + follow-ups in `contracts/identity/DECISION.md`.
