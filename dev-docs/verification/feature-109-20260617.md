# Feature #109 — iOS NFC canonical-locator normalization + recompute backfill + non-finite persistence guarding

Closes the #356 iOS obligation. Two WIs:

## Observations

- The headline finding: the **planned SchemaV9→V10 custom migration stage never
  fires** — V10 was model-set-identical to V9, and SwiftData keys migration on
  entity-shape hashes, so it can't distinguish the two. A disk test (seeding a
  persisted non-finite locator and asserting repair) is what exposed it; without
  that strengthened assertion the design would have shipped silently broken.
  Pivoted to a launch-time flag-gated backfill (`LocatorKeyBackfillMigration`).
- The on-device launch over a real existing library is the strongest evidence:
  the synchronous launch backfill ran to completion (flag set) over real
  persisted ReadingPositions without breaking startup or losing positions.
- WI-2's non-finite guard is a fault-injection path (a non-finite progression
  doesn't arise from normal device reading), so it's verified through the real
  `PersistenceActor` boundary by `PersistenceActorNonFiniteLocatorGuardTests`
  rather than a UI repro.
