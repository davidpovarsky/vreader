# Feature #111 — Android TXT reader — VERIFIED

The first Phase-3 capability under the #110 Android driver: plain-text reading at
parity. Verified on a real Android emulator, including a real 14MB UTF-16LE CJK book.

## Observations

- The offset/resume contract (`TxtDocument`) pins UTF-16 code units to the RAW
  decoded string (no line-ending normalization), so `charOffsetUTF16` stays exact on
  the real file's mixed CRLF/CR.
- TXT resume uses the LEGACY `VReaderLocator.wrapLegacy` + `ResumeResolver → Canonical`
  path (a TXT position has no Readium JSON), distinct from the EPUB `Precise` path.
- The reader chrome reuses the committed `vreader-reader.jsx` design (rule-51-compliant).
