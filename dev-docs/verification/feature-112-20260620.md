# Feature #112 — Android Markdown (.md) reader — VERIFIED

The second Phase-3 capability under the #110 Android driver: Markdown reading at parity,
a thin delta over the #111 TXT reader. Verified on a real Android emulator.

## Observations

- The instrumented render test deliberately drives the **library → routing → reader**
  path (launches `MainActivity`, taps the row) rather than launching the reader directly,
  so it also proves `BookFormat.md` routes correctly — not just that the renderer works.
- A real `.md` book does not exist in `test-books/` (the documented real-books-first
  exception: "no real MD today"), so synthetic fixtures are used. The decode / offset /
  resume paths are already real-book-verified via the #111 TXT path (identical code).
- `connectedDebugAndroidTest` auto-uninstalls the app after the run, so a static library
  screenshot isn't retained; the instrumented assertions (markers absent, heading present,
  resume landed) are the higher-fidelity end-to-end evidence and run on the real emulator.

## Known limitations (later #110 capabilities)

v1 is a SINGLE-LINE markdown subset over line chunks — fenced code blocks, multi-line
list items, continuation paragraphs, emphasis spanning a newline, tables, blockquotes,
links/images, `__bold__`, ordered lists, raw HTML, and thematic breaks render verbatim
(documented OUT of scope). AZW3 / PDF readers are the next reuse-leverage capabilities
under #110.
