# Feature #113 — Android backup-format contract model — VERIFIED

The Kotlin `@Serializable` DTO model for every WebDAV-backup archive section + manifest +
metadata (`contracts/identity/backup-format.md`, schema 3 / manifest 1), byte-compatible with
the iOS `BackupSectionDTOs.swift`, plus the cross-platform golden-vector conformance proving a
backup written on one platform restores on the other. Foundational tier (pure data layer +
contract) — no device/UI; the acceptance is the dual-platform conformance.

## Observations

- The conformance is **decode → re-encode → equal-to-the-shared-vector** on BOTH platforms.
  By transitivity, both agreeing with the same vector ⇒ they agree with each other — the
  interop proof. (Same pattern as the existing identity/locator conformance lane.)
- One real cross-platform subtlety surfaced + fixed during WI-3: iOS uses strict `UUID` id
  fields while Kotlin uses the wire `String`; the vectors use valid UUIDs so both decode. The
  Swift comparer was also hardened (Gate-4 Low) to distinguish JSON `Bool` from a numeric
  `NSNumber` (Foundation's `NSNumber(true) == NSNumber(1)`), matching Kotlin's `JsonElement`
  type-strict equality.
- The new Swift test file required an `xcodegen generate` so it registers in
  `vreader.xcodeproj/project.pbxproj` (a stale project ran 0 tests — a silent false-pass that
  was caught by checking the `.xcresult` test count, not just the RESULT line).
