# Feature #116 — Android WebDAV backup/restore backend — VERIFIED

The non-UI backend behind the #114 `BackupService` seam: a WebDAV client + content-addressed blob
store + backup collector + restore importer + the real `WebDavBackupService`, byte-for-byte the iOS
materializing-restore layout. Verified end-to-end by a **live** backup→wipe→restore round-trip on the
emulator against a real `rclone serve webdav` server.

## Observations

- **The live gate earned its keep.** The first connected run FAILED with
  `SAXNotRecognizedException: disallow-doctype-decl` — Android's `org.apache.harmony` SAX parser
  rejects the XXE feature flag that WI-1 had made *required* (fail-closed), so every PROPFIND threw on
  device. Robolectric/JVM (Xerces) never reproduced it. Fixed with a parser-independent fail-closed
  DOCTYPE ban + a fixed-UTF-8 `Reader` (closing a UTF-16 encoding bypass) + best-effort feature flags
  + a `resolveEntity` no-op; Codex audited the fix 3 rounds (`.claude/codex-audits/feat-feature-116-wi-6-roundtrip-audit.md`).
- On-device AndroidKeyStore AES-256-GCM `SecretCipher` worked first try (Robolectric couldn't test it).
- Atomic blob publish (`PUT .tmp` → `MOVE`) + PROPFIND-dedupe behaved correctly against rclone; no
  `.tmp` left behind.
