# Feature #117 — Android OPDS catalog backend — VERIFIED

The non-UI OPDS slice — feed parser + HTTP client + acquisition→import — verified end-to-end by a
LIVE round-trip on the emulator: fetch an OPDS feed over HTTP → parse → download the entry's EPUB →
import to the library, all against a local HTTP server serving a real EPUB.

## Observations

- The `SafeXml` helper (extracted from the #116 WI-6 lesson) carried the Android XXE hardening into
  OPDS for free — the DOCTYPE-rejected + UTF-16-not-a-bypass tests pass on Android's harmony parser.
- `run-opds-roundtrip.sh` served a real EPUB from `test-books/books/epub/` (real-books-first); it
  generates a minimal EPUB only if none is present.
- The acquisition magic-check (ZIP `PK\x03\x04` / `%PDF-`) + content-type guard means an HTML
  login/error page can never import under an `.epub` name.
