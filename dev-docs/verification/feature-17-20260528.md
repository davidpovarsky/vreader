## Observations

- CU works again this session (MCP screenshot captures real pixels) even though
  `system_profiler` reports a "Screen Sharing Virtual Display" — the display-state
  heuristic is a false negative; the empirical capture test is authoritative.
- The long-press-drag gesture is timing-sensitive: a ~0.8s hold before the drag
  is needed for PDFKit to enter selection mode (a faster drag pans/scrolls). The
  vreader highlight menu appeared on the well-formed long-press; a re-trigger with
  a shorter hold surfaced only the iOS system menu.
- The created highlight carries a real PDF anchor (`ZANCHORDATA`), so it restores
  on reopen (driver path already confirmed restore).
