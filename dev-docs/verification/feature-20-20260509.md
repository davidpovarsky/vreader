## Documentation finding (NOT a bug — tracker drift)

The `docs/features.md` row 20 description says `"Default" option in sort picker.` — implying an explicit menu item labelled "Default" or "Reset to default". The actual implementation has **no explicit "Default" item**: the picker shows the 4 enum cases from `LibrarySortOrder` (Title, Date Added, Last Read, Reading Time), and "reset to default" is achieved by re-selecting Title (the implicit default per `LibraryViewModel.swift:42`).

This is a documentation drift in the feature row, not a missing capability. The capability the row was tracking — "user can return the library to its default sort order without scrubbing UserDefaults / reinstalling the app" — works end-to-end via the picker. Updated feature row Notes accordingly.
