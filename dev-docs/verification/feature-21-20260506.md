## Summary

First device-verification round for feature #21 (paginated reading mode). Status was DONE-but-unverified since B04-B13 shipped. Bug #126's eval surface (PR #305, v3.14.13) made headless CSS-state inspection of the paginated EPUB possible.

**Verified** (this round, EPUB only, mini-epub3 fixture):

- The pagination style element (`<style id="vreader-pagination">`) is injected into the DOM after the EPUB loads.
- CSS multi-column layout is active in the body: `column-width: 362px` (matches `paginationCSS()` formula `viewportWidth − 40px columnGap`), `column-gap: 40px` (matches `EPUBPaginationHelper.columnGap` constant), `column-count: auto` (browser-computed from column-width), `overflow: hidden` (disables natural scroll, required for paged behavior).
- `document.documentElement.scrollWidth` matches `clientWidth` (980 px both) — for the small mini-epub3 chapter 1, the entire content fits on one column-page; pagination doesn't extend the scroll surface.

**Not exercised** (deferred):

- Multi-page navigation JS (`EPUBPaginationHelper.navigateToPageJS`) — the mini-epub3 fixture's chapter 1 fits on one page, so `scrollWidth > clientWidth` doesn't hold and the navigation primitive isn't observable. A larger fixture (or a wider page) would exercise this.
- TXT/MD pagination via TextKit containers — out of scope for this slice (TXT/MD use a different pagination path).
- PDF pagination via PDFKit — `PDFView` paged mode is a PDFKit feature, separately wired.

Per SCHEMA partial semantics, status stays DONE pending the multi-page navigation slice.

## Commands run

```bash
# Boot context: simulator booted; v3.14.16 installed.
DATA=$(xcrun simctl get_app_container booted com.vreader.app data | tail -1)
PLIST="$DATA/Library/Preferences/com.vreader.app.plist"

# Switch to paged mode via UserDefaults (no UI driving needed).
xcrun simctl terminate booted com.vreader.app
plutil -replace readerEPUBLayout -string paged "$PLIST"
xcrun simctl launch booted com.vreader.app

# Reset → seed → open mini-epub3
xcrun simctl openurl booted "vreader-debug://reset"
xcrun simctl openurl booted "vreader-debug://seed?fixture=mini-epub3"
KEY="epub:f284fd074ccd1d3c1a78985464d9e1be27975f4029f3c2ddef8428ca10684af4:2198"
xcrun simctl openurl booted "vreader-debug://open?bookId=${KEY}"

# Eval probes — each js_b64 is base64 of a CSS-state expression
for js_pair in \
  "style-present:document.getElementById('vreader-pagination') !== null" \
  "column-count:getComputedStyle(document.body).columnCount" \
  "column-width:getComputedStyle(document.body).columnWidth" \
  "column-gap:getComputedStyle(document.body).columnGap" \
  "overflow:getComputedStyle(document.body).overflow" \
  "scrollwidth:document.documentElement.scrollWidth" \
  "clientwidth:document.documentElement.clientWidth"; do
  bridge="${js_pair%%:*}"
  js="${js_pair#*:}"
  js_b64=$(printf "%s" "$js" | base64)
  xcrun simctl openurl booted "vreader-debug://eval?bridge=feat21-${bridge}&js=${js_b64}"
done
```

## Observations

- **The eval surface unblocked this verification**. Pre-bug-#126, paginated-mode-active was visually inspectable on a screenshot (column boundaries) but not programmatically observable. Now CSS state is read directly. Real-render-correlated check, not vibes.
- **Default reading layout is `.scroll`** (`ReaderSettingsStore.swift:58`) so this verification needed the paged mode toggled. Did so via direct plist write (`plutil -replace readerEPUBLayout -string paged`) — the same approach used in feature #44 round 8a (post-bug-#140) for switching `readerReadingMode`. Future: a `vreader-debug://prefs?key=...&value=...` command would clean this up; not blocking.
- **Single-page fixture limit**: mini-epub3's chapter 1 fits on one column-page in the iPhone 17 Pro viewport. To verify multi-page navigation (the `EPUBPaginationHelper.navigateToPageJS` primitive), a larger fixture would be needed. War-and-peace.txt isn't the right format. Filing as known gap, not a separate bug.
- **Codex audit not run**: this PR is verification + tracker update only, no Swift code change. Audit gate (`.claude/codex-audits/`) is required for Swift-touching PRs only.
