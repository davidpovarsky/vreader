## Observations

- **Regex escaping is real security work.** A user search query like
  `.*` could otherwise blow up the snippet or match unintended
  content; the test `regexSpecialCharsInQuery_escaped` is a
  regression gate against `NSRegularExpression` injection.
- **Run coalescing matters for visual quality.** Adjacent matches
  (e.g., search "cat" in "cat-cat") that would otherwise produce
  multiple bold runs with internal hyphens get merged into one bold
  run, producing cleaner visuals.
- **FTS5 tag stripping decouples the UI from the index format.** The
  snippet bytes that come out of SQLite FTS5 may contain
  `<mark>...</mark>` (or whatever delimiter); the highlighter is
  agnostic to that and recomputes its own ranges by matching the
  query string.
- **Pure-string transformation is the right architectural seam.** No
  filesystem, no SwiftData, no actor crossing — the function is
  isomorphic with `(snippet, query) → AttributedString`, easy to
  verify and impossible to drift due to side effects.
- **Production wiring confirmed by code-read.**
  `vreader/Views/Search/SearchResultRow.swift` consumes
  `HighlightedSnippet.build(snippet:query:)` and renders the result
  in a SwiftUI `Text` — the surface area where bugs could hide is
  thin.
