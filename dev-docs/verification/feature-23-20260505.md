print(f'fileByteCount: {d[\"fileByteCount\"]}')             # 14059220
prev = -1; bad = 0
for c in chapters:
    if c['globalStartUTF16'] <= prev: bad += 1

print(f'Non-monotonic: {bad}')                              # 0

## Observations

The Legado-rule auto-detection (bug #83 enabled 14 of 25 rules) produces a clean, exhaustive index for a real-world CJK web novel. Notable strengths:

1. **No false positives**: 1860 entries mirror the actual chapter count; nothing pollutes the index with random `章` mentions inside chapter bodies (e.g., "下一章" or "本章" in dialogue would have polluted a naive pattern).
2. **Multi-digit numeral parsing**: `第一千八百六十章` is decoded correctly. A regex that only handled `第N章` with single-digit Arabic numerals would miss every chapter past 9.
3. **Mixed spacing tolerance**: detector handles both `　` (Chinese typesetting standard) and regular space.
4. **Front matter inclusion**: `前言` is included as index 0 with a tiny 20-char span — useful for jumping back to the preface.

The 14 enabled rules likely cover: `第N章`, `第NN章`, `第NNN章`, `第NNNN章` (Arabic), `第Chinese章` (Chinese numerals 1-9), `第Chinese十Chinese章`, `第Chinese百Chinese章`, `第Chinese千Chinese章`, `前言` / `序章` / `楔子`, plus a few latin-script rules (`Chapter N`, `Section N`, etc.). The 11 disabled rules are likely the ambiguous ones from Legado's catalog (e.g., loose markdown-style headers, blank-line-bracketed titles) that were known to over-match in mixed-content texts.

**The verification gap (UI display + tap navigation)** is a tooling problem this iteration, not a feature gap. Computer-use's `screenshot` tool returns `SCContentFilter failure` (a macOS Screen Recording permission issue at the system level), and the installed app does not have `vreader-debug://` URL scheme registered (DebugBridge `Info.plist` URL types missing — separate gap), so I cannot drive TOC-button taps deterministically. The cached JSON is the durable, on-device evidence; the rendering and navigation legs are mechanical given a working harness. Recommendation: either fix the DebugBridge URL-scheme registration so future iterations can drive TOC taps via `simctl openurl`, or schedule the next /loop iteration when MCP screenshot recovers.
