## Blocker — bundled fixtures don't trigger TXT chapter detection

Round-2 attempted to drive the deferred UI gesture (TOCListView render + tap-to-navigate) on iPhone 17 Pro Sim (iOS 26.4) at v3.14.97. Two attempts:

**Attempt 1 — Position Test Book (`--seed-position-test`)**:
- Library shows the seeded TXT (~5 KB synthetic content).
- Reader opens to "Paragraph 1: This is test content for verifying reading position persistence…" (artifact: `feature-23-r2-position-book-no-chapters-20260509.png`).
- Annotations panel → Contents tab shows `No Table of Contents` / `No table of contents available for this document.` (artifact: `feature-23-r2-position-book-empty-toc-20260509.png`).
- **Why empty TOC is correct here**: `TestSeeder.generateTestText()` (`vreader/App/TestSeeder.swift:85-94`) produces the file as 100 lines of `Paragraph N: …` only — no `CHAPTER N` / `第 N 章` / `Section N` heading markers. The auto-TOC has nothing to match. The `Section N` strings I had observed earlier in search-result rows are search-classification labels, not in-text chapter markers.

**Attempt 2 — `vreader-debug://reset` then `vreader-debug://seed?fixture=war-and-peace`**:
- DebugBridge reset+seed succeeded (PR #427 confirmed the URL scheme registration via bug #121 fix); library shows only `war-and-peace` (the bundled fixture, 1708 bytes).
- Reader opens with the chrome showing the title `CHAPTER I` at the top (the auto-detected chapter heading is being rendered).
- Bottom pager shows `Previous   1/1   Next` (artifact: `feature-23-r2-warandpeace-chapter-i-but-1of1-20260509.png`) — **the auto-TOC sees this as ONE chapter**, not three, even though the fixture's text contains `CHAPTER I`, `CHAPTER II`, and `CHAPTER III` markers.
- Annotations panel → Contents tab shows the same `No Table of Contents` empty state (artifact: `feature-23-r2-warandpeace-empty-toc-20260509.png`).
- **Why the bundled war-and-peace fixture doesn't surface multiple chapters**: per the row-1 history, bug #83's fix only enabled 14/25 Legado regex rules to avoid false positives. The English `^CHAPTER\s+[IVXLCDM]+$` Roman-numeral rule appears to be among the disabled ones for this fixture's size + structure. The 14 MB CJK web novel that round-1 verified used Chinese-numeral chapter rules (`第一千八百六十章` etc.), which ARE enabled.

Both available bundled fixtures (`Position Test Book` synthetic + `war-and-peace.txt` 1708 bytes) therefore **cannot** drive the TOC render + tap-to-navigate UI gesture. The capability ships and works on real-world inputs (round-1 evidence), but verifying it through the simulator + DebugBridge harness would require either:

1. Bundling a chapter-marked TXT fixture in `vreader/Resources/DebugFixtures/` whose marker style matches an enabled Legado rule (e.g., a CJK fixture with `第N章`, or extending the enabled rules list to cover Roman-numeral English chapters with proper false-positive guards). OR
2. Driving a long-press / Files-app import of a real `.txt` (out of scope for autonomous verification — needs file-picker UI which simctl can't drive deterministically).

## Observations

- `vreader-debug://reset` + `vreader-debug://seed?fixture=war-and-peace` work end-to-end now (PR #427 / bug #121 fix). Round-1 noted "DebugBridge URL scheme not registered in Info.plist" as a deferral reason — that gap is closed; the new gap is fixture-content rather than scheme-registration.
- The reader does render `CHAPTER I` at the top of the war-and-peace book (so SOME chapter detection logic ran and identified at least one heading), but the TOC + pager treat the whole file as one chapter. The mismatch is interesting and might be worth a deeper look — flagged for future investigation, NOT a bug in this verification scope.
- The Annotations panel mechanics themselves (open from chrome, 4-tab segmented control, empty-state shape) are working correctly across both fixtures. That part is sound; only the TOC-population leg is gated.

## Recommendation

Add a small chapter-marked TXT to `vreader/Resources/DebugFixtures/` whose markers match an enabled Legado rule (CJK chapter pattern is the safest — already exercised at scale on the 14 MB book). A 5–10 KB fixture with 5–10 chapters would unblock TOC UI verification AND provide a target for future TOC-related XCUITest. This is feature-class work; tracked separately rather than as a bug.
