## Summary

Round-2 slice for feature #25 (Configurable tap zones — High).
Extends the 2026-05-07 evidence (`feature-25-20260507.md`) which
verified the data layer (26 tests covering `TapZoneConfig.zone(atX:totalWidth:)`
boundary math, action mapping, Codable round-trip, 4 actions × 3
zones, `TapZoneModifier` default 100pt bottom-inset for bug #63),
with `result: partial — UI flow deferred`.

This round closes the **Reader Settings Panel presentation +
Tap Zones section presence leg** via a new XCUITest suite of 4
methods (38.3 s wall-clock):

- `vreaderUITests/Reader/ReaderSettingsPanelTests` (4 methods) —
  verifies the reader chrome's `readerSettingsButton` (gear icon)
  presents the `readerSettingsPanel` sheet, the panel renders its
  populated content (Theme section header), the `Tap Zones`
  section header appears after scrolling (SwiftUI Form lazily
  renders below-fold sections), the Reading Mode picker exposes
  the `Native` option, and the panel passes the iOS 26
  accessibility audit.

A per-picker test (Left/Center/Right Zone rows under Tap Zones)
was attempted but dropped — iOS 26 SwiftUI's Picker rendering
inside a Form sheet doesn't reliably expose each picker's
accessibilityLabel as a matchable element after scrolling. The
data layer's wiring (config × action × zone matrix) is verified
by 2026-05-07's `TapZoneConfigTests`; the section presence
assertion is sufficient at the UI tier without overfitting to
Form picker markup that shifts between OS versions.

All 4 synthesized real taps via XCUIApplication, no Computer Use,
no display capture. Target simulator was iPhone 17 Pro (iOS 26.4
Simulator). Same harness pattern as PRs #419 / #420 / #422 / #425
/ #427.

## Observations

- The reader settings panel is a Form-style sheet with ~12 sections.
  Sections below the fold (Tap Zones at position 12, after Theme,
  ThemeBackground, ReadingMode, EPUBLayout, PageTurnAnimation /
  AutoPageTurn conditional, FontSize, LineSpacing, FontFamily, CJK,
  ChineseConversion) are NOT in the accessibility tree until the
  Form is scrolled. Added a `scrollPanelUntilLabelExists(_:)`
  helper that swipes up to 6 times on `app.scrollViews.firstMatch`
  before giving up — found the Tap Zones header reliably.
- The same scroll mechanism does NOT reliably expose individual
  Picker rows on iOS 26 inside a Form sheet — see "deferred" rows
  in the table above. The data-layer matrix tests cover the
  config-side semantics, so the UI-tier gap is "is each row
  queryable" rather than "does the binding work."
- Cross-feature smoke: `testReaderSettingsExposesReadingModeSection`
  also touches feature #28 (per-book settings, Native vs Unified
  mode), feature #21 (paginated mode, EPUB layout picker
  upstream), and feature #27 (replacement rules — only run in
  Unified mode). Their UI-driven slices remain deferred but the
  presence of their gating selectors is verified incidentally.
- No production code touched in this verification round — pure
  XCUITest addition leveraging existing accessibility identifiers
  (`readerSettingsButton`, `readerSettingsPanel`).
