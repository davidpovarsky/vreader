## Summary

Round-2 verification of feature #26 (TTS read aloud). Closes the
deferred **UI start/stop gestures + audio playback** legs from
round-1 (`feature-26-20260507.md`).

Round-1's 101-test data layer slice (TTSService state machine,
HTTPTTSProvider config/cache, FoliateTTSAdapter JS bridge) is
authoritative for the underlying state machine + provider correctness.
Feature #41 round-2 (`feature-41-20260509.md`) already verified the
**TTS Start** gesture and the **TTS bar visibility** with auto-scroll
on the same fixture. This round-2 closes the remaining UI-driven gap:
the **Pause/Resume cycle** and explicit **Stop teardown**.

End-to-end pass on iPhone 17 Pro Sim (iOS 26.4) at v3.14.111
(commit 239ac16) against bundled `war-and-peace.txt`:

- Tap speaker icon → TTS bar appears with ⏸ Pause / ◾ Stop / "1.2x"
  speed slider; view auto-scrolls to CHAPTER I as narration begins
  (TTS auto-scroll, feature #41 working).
- Tap **Pause (⏸)** mid-narration → bar swaps the left button to
  ▶ Play (paused state). Auto-scroll halts at the position the
  narrator paused. Stop button + speed slider remain interactive.
- Tap **Resume (▶)** → bar swaps back to ⏸ Pause. Narration resumes
  from the paused position; view continues auto-scrolling forward
  through CHAPTER II → CHAPTER III content.
- Tap **Stop (◾)** → TTS bar disappears entirely; reader's normal
  bottom toolbar restored (progress bar showing 56%, pager
  "Previous | 1/1 | Next", "56% / 2m read" reading-time pill).
  Final visible content includes "End of synthetic fixture excerpt."

The state-machine transitions match the round-1 test contract:
`.idle → .speaking → .paused → .speaking → .idle`. UI button swaps
correlate with state changes. Stop teardown removes the bar and
restores chrome correctly.

## Observations

- **Pause is instantaneous in the simulator.** No audible "tail" or
  delayed silence — the AVSpeechSynthesizer halts on the next utterance
  boundary.
- **Resume picks up at the paused offset, not from the beginning.**
  Visible scroll position is preserved across the pause window, and
  the next narration boundary continues from the same content. This
  matches the round-1 `TTSServiceTests` test for resume-from-position.
- **Stop is destructive and final.** Unlike Pause, Stop removes the
  TTS bar entirely — the user must tap the speaker icon again to
  restart, which begins narration from the current scroll position.
  This is the documented `.idle` state behavior.
- **Auto-scroll progress as inferential evidence of audio playback.**
  Round-1 noted the simulator can play audio via Mac speakers, but
  capturing audio is out of scope for screenshot-based evidence. The
  view auto-scrolling 0% → 56% across the Pause/Resume cycle is
  inferential proof that `AVSpeechSynthesizerDelegate.willSpeakRange`
  callbacks are firing — those callbacks are the only path that
  drives `coordinator.updateHighlight(offset:)` →
  `uiState.scrollToOffset` → `UITableView.scrollToRow` (per round-2
  of feature #41's code-read at `TTSHighlightCoordinator.swift:66`).
  Auto-scroll is therefore proxy evidence for active audio playback.
- **Speed slider remains interactive in all three TTS states.** The
  "1.2x" indicator + slider visible across all 4 screenshots — could
  be exercised in a future round-3 (mid-narration speed change). Not
  blocking VERIFIED.
- **No new bugs surfaced.** This is a clean round-2 close-the-loop for
  the TXT slice of feature #26.
