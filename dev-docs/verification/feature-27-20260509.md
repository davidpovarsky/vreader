## Summary

Round-2 verification of feature #27 (Content replacement rules)
attempted the deferred real-render verification leg from
`feature-27-20260507.md` (round-1 result=partial). The plan was:
add a `Pierre` → `Peter` replacement rule, switch to Unified mode
(the only mode where rules apply per bug #128 banner), and confirm
the rendered text shows "Peter" instead of "Pierre" against the
bundled `war-and-peace.txt` fixture.

**Result: blocked.** The Unified reading mode for TXT is broken
in compound ways (truncation, missing chrome, toggle-blank), and
Pierre's location in the fixture (line 16, in CHAPTER III) is below
the truncation point that Unified mode renders. I could not reach
the visible Pierre/Peter to verify the rule application.

Filed as **bug #158 / GH #468** (Reader/* Medium). Per verify-cron
scope, this round files but does not fix.

Native mode behavior was confirmed independently: at ~47% scroll
position, "Pierre" renders as "Pierre" (rule does NOT apply in
Native mode) — this matches bug #128's documented native-mode
no-op behavior, which is why the cheap-path banner was shipped in
the first place. So Native is "working as designed (broken)";
Unified is "broken differently".

## Observations

- **The "rule active" state is correct.** Replacement Rules screen
  confirms the Pierre → Peter rule is saved + enabled (green toggle,
  Global tag). The rule should be applied by the Unified pipeline
  per the production wiring at `ReaderContainerView.swift:194-197`.
  But the Unified renderer never reaches the line where Pierre
  appears, so application is unobservable.
- **Search engine sees the full document.** Searching "Pierre" in
  Unified mode returns 1 hit with snippet `... ordered her carriage
  and now departed. Pierre, who was alone among the guests not
  inv... — Section 8`. So the source text is fully indexed; only the
  Unified renderer's viewport is incomplete. This is bug #158.
- **Native rendering of Pierre is correct.** At ~47% scroll position
  in Native mode, "Pierre" appears verbatim — confirming the bug
  #128 native-mode no-op is still in effect (rules don't apply in
  Native), which is the entire reason the banner exists.
- **Unified mode's Reading Settings footer** says "Unified reflow
  engine is coming in V2." This signals Unified is WIP — but the
  toggle exists and produces a partial render, which is worse UX
  than hiding it. Bug #158 proposes either a capability-gate (cheap
  fix) or making Unified actually work (feature-class scope).
- **No regressions surfaced** in the data-layer slice. Round-1's 14
  `ReplacementTransformTests` are still authoritative for the
  transform pipeline correctness.
