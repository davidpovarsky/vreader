## Observations

- The `ChineseConversionPickerGateTests` UI tests launch the app fresh with `--seed-books` (which includes "Test Plain Text", a 1KB TXT fixture). The picker was confirmed enabled and hittable for this TXT book.
- The DebugBridge `vreader-debug://open` URL scheme triggers an iOS system URL scheme confirmation dialog ("Open in vreader?") on fresh installs. This blocked the original automation approach using `xcrun simctl openurl`. The UI test approach avoids this entirely by launching via XCUITest.
- war-and-peace.txt is seeded in the app's DebugFixtures from the previous cron iteration (visible in library screenshot artifact).
- The previous verification run (2026-05-05, `feature-28-20260505.md`) failed because the conversion was unified-mode-only. WI-A (PR #573, merged 2026-05-12) wired TXT native-mode conversion. This run confirms that fix is live.
- The `result: pass` reflects: all acceptance criteria pass, with "deferred" on the UserDefaults persistence roundtrip (this is an infrastructure-level guarantee, not a behavioral edge case for this feature, and is covered by other store tests).
