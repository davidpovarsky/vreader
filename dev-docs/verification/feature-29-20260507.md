## Why partial, not pass

Per `dev-docs/verification/SCHEMA.md`:

> Live integration tests against the actual backend counts as
> verification. Unit + protocol tests alone are NOT verification.

The 135 passing tests give **strong Gate-4-class evidence** across
the WebDAV backup stack:

- **`BackupViewModel` end-to-end** — production ViewModel exercised
  through state machine: idle → backing up → completed; idle →
  listing → list rendered; idle → restoring → restored. URLSession
  is mocked; everything above is real.
- **WebDAV protocol surface** — `WebDAVClient` covers all the verbs
  (PROPFIND, MKCOL, PUT, GET, MOVE, DELETE) the backup feature
  uses. Auth header construction, redirect handling, and error
  mapping are tested directly.
- **ATS exception is locked by test** — the
  `NSAllowsArbitraryLoads: true` Info.plist key (bug #110 fix) is
  asserted in `WebDAVATSTests`. The README's note about
  Tailscale-fronted hosts depends on this being preserved.
- **ZIPWriter round-trip** — backup archives are verified to round-
  trip including unicode filenames + large data, and to error
  cleanly on missing entries (which would otherwise corrupt
  selective restore).
- **BlobPath canonical-key + manifest parity** — the
  fingerprint-keyed paths in `BlobPath` line up with the manifest
  entries in `BackupLibraryManifest`. Loss of parity here would
  produce silent restore failures (the bug-class that drove
  feature #46's existence). Both directions are tested.
- **The live device-verify already happened** — the row records
  "Manually verified end-to-end on iPhone 17 Pro simulator:
  credentials → Back Up Now → list → Restore → Delete all work
  against a real Bytemark/Apache WebDAV server" at PR #128 ship
  time. That stands as the Gate-5-class evidence; this slice adds
  the audit-gate (Gate 4) test catalog.

What's deferred: the `WebDAVBackupIntegrationTests` suite that
runs against a live HTTP WebDAV endpoint behind
`WebDAVIntegrationConfig.isEnabled` is intentionally not exercised
this iteration (no live server harness available). Re-running
that suite against a live server (rclone serve / Apache) would
give Gate-5 evidence in the test catalog itself.

`result: partial` per SCHEMA. Tracker status stays `DONE`.

## Observations

- **The "99 unit tests" line in the row was the count at PR #128
  ship time.** Today the core backup stack has **135 tests across
  11 suites**, growing as features #46 (materializing restore) and
  #47 (lazy download) shipped against the same core protocols.
  The materializer + lazy-download tests are NOT counted here —
  they're verified separately on their own feature rows. This
  slice is the original-feature scope only.
- **`BackupProviderContract`** is the protocol-conformance test
  that all backup adapters must pass. WebDAVProvider is the only
  shipping adapter today, but the contract tests guard against
  future iCloud / S3 adapters silently breaking the spine.
- **Backup is one of the few features exercised against a real
  network boundary in CI gating** — the
  `WebDAVBackupIntegrationTests` suite is gated by
  `WebDAVIntegrationConfig.isEnabled` so a developer with a local
  rclone WebDAV running can exercise it without the suite
  blocking everyone else's tests. That gating is the right
  design but means slice verification can't run it without
  the harness.
- **ATS configuration is a tested invariant** — `WebDAVATSTests`
  asserts the `NSAllowsArbitraryLoads: true` Info.plist key still
  exists. If a future PR strips this for "security cleanup", the
  test fails immediately. Bug #110's fix is locked in.
- **Manifest + projection are the restore spine.** The
  manifest-and-projection test pairs validate the data shapes
  that determine whether restore can find each book's blob and
  match it to the SwiftData record. This is where feature #46's
  materializing restore depends on feature #29's manifest format
  being stable.
- **The backup ZIP format handles unicode filenames** —
  `createArchive_unicodeFilenames_supported` is the test that
  guards CJK book titles + diacritic filenames from corrupting
  the archive directory record.
