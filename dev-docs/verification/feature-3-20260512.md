# Feature #3 — Manual text highlighting — round-2 device verification (EPUB)

Round-1 (`feature-3-20260507.md`) closed the data-side contract with 84 tests across 14 suites (ViewModel state machine + coordinator + bridge + dedupe + anchor + snippet + persistence + migration). UI-gesture leg ("select text → tap Highlight → see overlay") deferred — needed display.

This round (2026-05-12, v3.17.2) drives the gesture end-to-end on EPUB via CU now that the SCContentFilter outage is resolved. **TXT and MD/PDF formats are NOT exercised this round** — TXT chapter-mode highlighting is blocked by the WI-7 chapter-local↔global offset deferral (see bug #160 + feature #48), and no MD/PDF fixtures are bundled in `DebugFixtures/`. The EPUB leg is the largest cross-format risk (custom WKWebView bridge, JS highlight API, CSS Highlight overlay) and is the primary "needs display" deferral that the round-1 evidence cited.

## Commands run

```bash
SIM_ID=$(xcrun simctl list devices booted | awk '/Booted/ {print $(NF-1); exit}' | tr -d '()')
DATA=$(xcrun simctl get_app_container $SIM_ID com.vreader.app data)

# Fresh state
xcrun simctl terminate $SIM_ID com.vreader.app
xcrun simctl launch $SIM_ID com.vreader.app
xcrun simctl openurl $SIM_ID "vreader-debug://reset"
xcrun simctl openurl $SIM_ID "vreader-debug://theme?mode=light&fontSize=18"
xcrun simctl openurl $SIM_ID "vreader-debug://seed?fixture=mini-epub3"

# Open the EPUB
BOOK_KEY="epub:f284fd074ccd1d3c1a78985464d9e1be27975f4029f3c2ddef8428ca10684af4:2198"
ENCODED=$(printf '%s' "$BOOK_KEY" | sed 's/:/%3A/g')
xcrun simctl openurl $SIM_ID "vreader-debug://open?bookId=$ENCODED"
xcrun simctl openurl $SIM_ID "vreader-debug://settle?token=open"

# Snapshot pre-highlight (highlightCount=0)
xcrun simctl openurl $SIM_ID "vreader-debug://snapshot?dest=pre.json"
cat "$DATA/Library/Caches/DebugBridge/pre.json" | grep highlightCount  # 0

# CU drives the gesture:
#   1. mouse_move (130, 190)   ← positions cursor on word "emphasized"
#   2. left_mouse_down
#   3. wait ~1.5s (exceeds iOS long-press threshold ~500ms)
#   4. left_mouse_up
#   → Custom Text-Selection menu appears with Highlight / Add Note / Copy
#   5. left_click (185, 380)  ← taps "Highlight"

# Snapshot post-highlight (highlightCount=1)
xcrun simctl openurl $SIM_ID "vreader-debug://snapshot?dest=after.json"
cat "$DATA/Library/Caches/DebugBridge/after.json" | grep highlightCount  # 1

# Close (CU left_click on back chevron) + re-open
xcrun simctl openurl $SIM_ID "vreader-debug://open?bookId=$ENCODED"
xcrun simctl openurl $SIM_ID "vreader-debug://settle?token=reopen"

# Snapshot persistence (highlightCount=1)
xcrun simctl openurl $SIM_ID "vreader-debug://snapshot?dest=reopen.json"
cat "$DATA/Library/Caches/DebugBridge/reopen.json" | grep highlightCount  # 1
```

## Observations

1. **CU long-press now works reliably.** The pattern is: `mouse_move → left_mouse_down → wait 1.5s → left_mouse_up`. Earlier in the session the same pattern produced unwanted text selection during bug-#167 verification (because we wanted a swipe/pan, not a long-press). Here long-press is the target, and the same iOS behavior makes it reliable. Tool-call latency between `mouse_down` and the first `mouse_move` exceeds the long-press threshold; not moving at all guarantees long-press recognition.
2. **Custom edit menu shows both the system "Copy / Look Up / Translate" and vreader's "Highlight / Add Note / Copy"** — the latter is the custom `UIMenu` registered by bug #44 plus subsequent additions. The visible side-by-side rendering confirms the menu hierarchy is wired correctly.
3. **CSS Highlight overlay re-renders correctly after re-open** — yellow paint appears on the originally-highlighted word on the first body render after navigation, without a perceptible flash of unstyled text. The CFI-anchor restoration path is intact.
4. **Read time tracker advanced** (`Chapter One / 1m read` → `2m read`) during the gesture sequence, confirming the reading-session tracker (feature #41) still ticks while a CU-driven gesture is active.

## Cross-format scope

This round verified EPUB only. TXT, MD, PDF paths covered as follows:
- **TXT (chapter mode)** — Blocked by WI-7 chapter-local↔global offset translation (feature #48). Chapter-mode highlights store chapter-local offsets as if global. Not a feature #3 regression — known scope gap with its own tracking row.
- **TXT (continuous mode)** — No continuous-mode TXT fixture bundled in `DebugFixtures/`. Round-1's 84 unit tests cover the TXT path at the ViewModel + UTF-16 invariant layer.
- **MD** — No MD fixture bundled. Round-1 unit tests cover the path.
- **PDF** — Round-1 unit tests cover; PDFKit's PDFAnnotation rendering is a separate code path from `HighlightableTextView` and the WKWebView CSS Highlight overlay.
