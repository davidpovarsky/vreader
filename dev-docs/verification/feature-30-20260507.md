## Observations

- **Two complementary slices give strong confidence.** The unit
  tests prove the contract; the live end-to-end observation from
  #43 round-3 proves production reaches that contract correctly.
- **Path sanitization matters.** Fingerprint keys are
  `<format>:<sha256>:<bytes>` â€” both colons (Unix path safe) and
  slashes (defense in depth) are sanitized to underscores so the
  filename `azw3_<sha>_<bytes>.jpg` is portable across iOS
  filesystems.
- **Auto-extracted vs user-picked covers share the same store.**
  The "CustomCovers" name is slightly misleading â€” it covers BOTH
  manual user-picked covers (via `PhotosPicker`) AND
  auto-extracted ones from EPUB OPF / AZW3 EXTH metadata. The
  store is a pure key-value blob keyed by fingerprint; first
  writer wins (`BookImporter` only auto-saves when
  `!CustomCoverStore.hasCover(for: fingerprintKey)` per
  `BookImporter.swift:180`), so a user-set cover is never
  overwritten by re-import.
- **Image resizing has a max-dimension cap.** Large images are
  resized to fit memory budgets; small images aren't upscaled
  (preserves cover quality for low-res sources). Two tests cover
  both branches.
