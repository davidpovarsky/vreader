## CU-driven repro on iPhone 17 Pro Sim (iOS 26.4) at v3.14.84

1. `xcrun simctl install "iPhone 17 Pro" <Debug-iphonesimulator>/vreader.app`

12. **Disk verification**: JPEG present at `‚Ä¶/Library/Application Support/CustomCovers/epub_<sha>_102400.jpg` ‚Äî baseline JPEG, JFIF + Exif, 512√ó384, 117,819 bytes.

## Observations

- **No regressions** ‚Äî round 1's data-layer guarantees still hold;
  this round only ADDS coverage on the gesture path.
- **`PhotosPicker`'s "Private Access to Photos" message is the
  iOS 14+ default scoped-access flow** ‚Äî the user grants access to
  individual selected items, not the full library. vreader doesn't
  request `NSPhotoLibraryUsageDescription` for full-library access,
  which is the right call (user only ever needs ONE image at a time
  for a cover).
- **Auto-resize verified by output dimensions**. The picked photo
  was downsized to 512√ó384 (matches `CustomCoverStore`'s configured
  max-dimension cap from round 1's `setCover_resizesLargeImage`
  test). Production confirms the data-layer test's contract.
- **EPUB fingerprint sanitization works in production**. The
  on-disk filename `epub_<64-hex-sha>_102400.jpg` confirms colons in
  the canonical fingerprint key (`epub:<sha>:<bytes>`) get replaced
  with underscores by `CustomCoverStore.coverPath(for:)` ‚Äî round 1's
  `coverPath_sanitizesColons` test contract holds at runtime.
- **Cross-launch persistence** confirms the cascade chain works
  forward: `setCover` ‚Üí disk write ‚Üí next-launch `getCover` ‚Üí grid
  re-render. The reverse cascade (delete book ‚Üí removeCover) is
  still passes-by-inspection (not exercised in this round) ‚Äî the
  unit test `removeCover_deletesFile` + the code-read at
  `PersistenceActor+Library.swift` are sufficient evidence.
- **Result: `pass`** ‚Äî both deferred slices from round 1 confirmed
  end-to-end. Status moves to `VERIFIED`.
