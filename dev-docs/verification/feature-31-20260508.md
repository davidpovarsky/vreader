## CU-driven repro on iPhone 17 Pro Sim (iOS 26.4) at v3.14.91

1. `xcrun simctl install "iPhone 17 Pro" <Debug-iphonesimulator>/vreader.app`
2. `xcrun simctl terminate "iPhone 17 Pro" com.vreader.app`
3. `xcrun simctl launch "iPhone 17 Pro" com.vreader.app --uitesting --seed-position-test --reset-preferences`
4. Open Position Test Book ŌåÆ reader (TXT, 100 paragraphs, ~10 KB).

### Reading Settings: EPUB Layout switch reveals Auto Page Turn UI

5. Tap **AA** toolbar ŌåÆ Reading Settings panel slides up at medium detent.
6. Scroll to **EPUB Layout** segmented picker (currently on `Scroll`).
7. Tap **Paged** ŌåÆ segmented control flips. Captured at `feature-31-r2-paged-mode-20260508.png`.
8. Scrolling further reveals the **Page Turn Animation** segmented picker (`None`/`Slide`/`Cover`) AND the **Auto Page Turn** toggle row (default OFF). Captured at `feature-31-r2-auto-pagetune-section-20260508.png`.

### Auto Page Turn toggle (sustained press required)

9. **Toggle Auto Page Turn ON** ŌĆö required `mouse_down + 0.4s wait + mouse_up` (instant `left_click` did NOT register, mirroring the same SwiftUI Toggle quirk documented in feature #32 round 2). The toggle flips green; an **Interval slider** appears below with default `5s`. Captured at `feature-31-r2-auto-page-turn-on-20260508.png`.
10. Toggle OFF (sustained press): the **Interval slider disappears**, leaving only the toggle and footer "Automatically turn pages at the set interval. Pauses on user interaction." Captured at `feature-31-r2-toggle-off-no-slider-20260508.png`.
11. Toggle ON again: slider re-appears at the previously-set value (state preserved within session).

### Settings persistence across panel re-open

12. Dismiss the panel by tapping outside its sheet, then tap **AA** again to re-present.
13. Scroll to the relevant section: **Paged** still selected on the EPUB Layout picker, **Auto Page Turn** still ON, slider still at 5s. The animation picker (None/Slide/Cover) is also visible because Paged is the active mode. Captured at `feature-31-r2-settings-persisted-20260508.png`.

### Live page advancement ŌĆö gated by fixture pagination

14. Dismiss panel; reader displays Position Test Book. Bottom navigator shows **`< Previous   1 / 1   Next >`** ŌĆö the 100-paragraph, ~10 KB body fits within `NativeTextPaginator`'s single-container layout at `UIScreen.main.bounds.size` (393├Ś852pt) with default 18pt font.
15. Wait 6 seconds with auto-page-turn ON at 5s interval.
16. **Observed**: still page 1 of 1; no advancement. Captured at `feature-31-r2-after-6s-still-page1of1-20260508.png`.

The expected behavior at this state is "no advancement" because of
the at-last-page short-circuit in
`AutoPageTurner.scheduleTimer`:

```swift
if nav.currentPage >= nav.totalPages - 1 {
    self.stop()
    return
}
nav.nextPage()
```

For `totalPages == 1`: `0 >= 0` ŌåÆ `stop()` returns; `nextPage()`
never runs. This is correct, defensive behavior, not a regression.

### Why the fixture produces 1 page

`TextReaderUIState.updatePagination` calls
`paginator.paginateAttributed(attributedText:, viewportSize: UIScreen.main.bounds.size)`.
On iPhone 17 Pro that's 393├Ś852pt. With the default 18pt font, NSLayoutManager
fits ~5000 chars in one container, which exceeds the Position Test
Book's body length. To produce >1 page in this fixture would require
either (a) a larger fixture, (b) a smaller viewport (passing the actual
text-view rect rather than the full screen), or (c) bumping the font
size dramatically. None of these change the verified surface; they
would only let live advancement be visually observed.

## Observations

- **CU+iOS Toggle quirk (third encounter)** ŌĆö same `mouse_down + 0.4s wait + mouse_up` pattern needed as feature #32 (Custom Background) and feature #25 (Tap Zones) round 2. SwiftUI's Toggle uses a tap-with-press-confirmation gesture recognizer; CU's instant `left_click` doesn't satisfy it. Documenting again so future iterations don't waste cycles. The neighboring segmented pickers (Native|Unified, Scroll|Paged, None|Slide|Cover) accept instant clicks fine.
- **Fixture limit, not a bug**: The Position Test Book's ~10 KB body paginates to a single `NativeTextPaginator` page at 393├Ś852pt viewport with 18pt font. This is correct pagination ŌĆö at that font + viewport size, the entire body fits in one container. The `AutoPageTurner` then correctly stops via `currentPage >= totalPages - 1`. To exercise live advancement on this fixture, either (a) shrink the viewport to actual text-view rect (not full screen), (b) bump font size to 28-32pt, or (c) introduce a much larger TXT/MD fixture (e.g., a 100KB+ fixture with chapter index). None affect the wiring already verified here.
- **Why this matters even partial**: Round 1 covered the 14-test state machine. The ONLY thing this round adds is "the toggle, the slider, and the conditional-reveal logic actually work in the rendered UI" ŌĆö and that part is now confirmed end-to-end. The state-machine tests already cover what `AutoPageTurner.start` does internally; the verification gap was strictly "does the gesture path reach `start`" which this round closes.
- **Animation picker visible but un-exercised**: `None | Slide | Cover` segmented picker shows when Paged is active. Live transition rendering between pages is also gated by the same multi-page-fixture limit; round-1 unit tests cover the per-style `CAAnimation` shape generation directly.
- **Result: `partial`** ŌĆö the UI surface gains coverage; live advancement remains deferred behind a documented, non-bug fixture constraint. Status stays `DONE`.
