## Summary

Round-2 attempt at the deferred UI-gesture leg from
`feature-31-20260507.md` ("End-to-end UI: enable auto page turn →
reading view advances every N seconds"). Driving against a real EPUB
on the iPhone 17 Pro simulator surfaced a **wiring bug** that the
round-1 14-test data-layer slice didn't reach: the Auto Page Turn
toggle in the settings panel has zero effect for EPUB / PDF / AZW3 /
MOBI readers because no host listens to `store.autoPageTurn` or
instantiates an `AutoPageTurner` for those formats. Only TXT and MD
wire it.

Filed as **bug #156 / GH #456**. Per the verify-cron scope, this
round files but does not fix.

## Observations

- **Auto Page Turn toggle visibility**: visible in settings panel for
  EPUB regardless of format support. The toggle and interval slider
  appear when `store.epubLayout == .paged`, which gates by *layout
  preference* not by *renderer integration*. This is the root of the
  bug — settings UI promises a feature that the EPUB renderer doesn't
  implement.
- **Side-effect of round-2 driving**: I left the EPUB in Paged layout
  + Auto Page Turn ON in the simulator's persisted settings (the user
  toggle persists to UserDefaults). Subsequent verify rounds may want
  to reset via `--reset-preferences` to start clean.
- **Per-book settings carry-over**: feature #37's earlier round-2 had
  toggled "Custom settings for this book" ON for the mini-epub3
  fixture; that state persisted across this round's relaunches. Not
  a bug — just a note for verification reproducibility.
- **No regressions observed** in the unit-test layer. Round-1's 14
  tests are still authoritative for the data layer's correctness; the
  bug is purely at the integration seam.
