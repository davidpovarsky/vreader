## Why partial, not pass

Per `dev-docs/verification/SCHEMA.md`:

> **Live integration tests against the actual backend** counts as
> verification. Unit + protocol tests alone are NOT verification —
> they cover the audit gate (Gate 4), not the integration gate
> (Gate 5).

Three of the 19 tests **do** count as Gate-5-class integration
because they hit the real system backend:

- `canLookUp_returnsTrue_forCommonEnglishWord` — calls
  `UIReferenceLibraryViewController.dictionaryHasDefinition(forTerm:)`
  which is the real Apple system dictionary API. Returns true for
  "hello" (or whatever common English word the test uses) only if
  the system dictionary is available on this iOS version.
- `canLookUp_returnsFalse_forGibberish` — same API, opposite branch.
- `viewController_createsForWord` — instantiates the real UIKit
  `UIReferenceLibraryViewController` with a word; passes if the
  modal can be created (which depends on the dictionary being
  installed).

What's deferred: the **gesture flow** — selecting text in the reader
WebView/UITextView and routing through `UIMenuController` /
`UIEditMenuInteraction` to the Define handler. That's purely UI
event plumbing; the data layer + system-API integration are
covered.

`result: partial` per SCHEMA. Tracker status stays `DONE`.

## Observations

- **Selection parsing is exhaustively tested.** 13 of 19 tests cover
  `extractWord_fromSelection` across English / CJK / mixed / emoji
  / punctuation / whitespace / newlines / tabs / single-space.
  The "first-word-only" contract is enforced explicitly.
- **System dictionary integration is real.** Three tests hit
  `UIReferenceLibraryViewController` directly without mocks — that's
  rare in unit suites because the modal-presenter API isn't usually
  unit-test friendly. The fact that these tests run AT ALL (and
  pass) means the simulator has the system English dictionary
  installed and reachable.
- **CJK handling matters because Chinese/Japanese readers won't
  have spaces between words.** The "single CJK character" and
  "mixed CJK + English" paths cover the realistic case where the
  reader extracts a single character or the leading character
  cluster from a CJK selection.
- **Translate is feature #18's territory.** Dictionary's role here
  is the menu entry + AI translate sheet routing; the actual
  translation API call is in `AIService` covered by feature #18
  evidence.
