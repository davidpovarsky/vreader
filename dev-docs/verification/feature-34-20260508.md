## Summary

Round-2 slice for feature #34 (Collections / tags / series organization
— Medium). Extends the 2026-05-07 evidence (`feature-34-20260507.md`)
which verified the data layer at SchemaV6 + `PersistenceActor` boundaries
(19 `CollectionPersistenceTests` + 11 model tests covering create /
rename / delete / membership / cascade / Unicode-CJK / dup-rejection)
with `result: partial — UI flow deferred`.

This round closes the **Collections sidebar UI presentation leg** via
a new XCUITest suite of 6 methods (49.5 s wall-clock):

- `vreaderUITests/Library/CollectionSidebarTests` (6 methods) —
  verifies the sidebar opens from the `collectionsToolbarButton`,
  presents under `Filter` navigation title, exposes the `All Books`
  filter row, exposes `New Collection` entry point, surfaces the
  inline `newCollectionTextField` + `addCollectionButton` (with the
  Add button correctly disabled on empty input — gating against
  whitespace-only collection names), dismisses cleanly via Done, and
  passes the iOS 26 accessibility audit on the empty-state shape.

The sidebar is a sheet-based filter UI that gates downstream
collection / tag / series browse and assignment. With `seed: .books`
the library has fixture rows but zero collections, so the sheet
renders in its empty-state shape (no Collections / Tags / Series
sections beyond the entry-point row). Per-collection assignment
(context menu → Add to Collection) and filter-application (tap a
collection → library filters down) require seeded collections; those
are still deferred — they need a follow-up suite that pre-seeds
collections rather than running on empty state.

All 6 synthesized real taps via XCUIApplication, no Computer Use,
no display capture. Target simulator was iPhone 17 Pro (iOS 26.4
Simulator). Same harness pattern as PRs #419 (#6 round-2), #420
(#9 round-2), and #422 (#1 round-2).

## Observations

- The `addCollectionButton` is correctly disabled on empty input — a
  small but important gate against whitespace-only collection names
  that would muddy the persistence layer's case-insensitive
  duplicate detection. Confirmed by the test pinning
  `XCTAssertFalse(addButton.isEnabled)` after the inline TextField
  appears.
- `seed: .books` produces 8 fixture books but zero collections, zero
  tags, zero series. Tags / Series sections are conditionally
  rendered (`if !allTags.isEmpty`), so this round only exercises the
  empty-state shape. A follow-up suite that seeds collections (and
  optionally tags via `Book.tags` writes) is the next slice toward
  flipping #34 to `VERIFIED`.
- No production code touched in this verification round — pure
  XCUITest addition leveraging accessibility identifiers already
  present on the production UI.
