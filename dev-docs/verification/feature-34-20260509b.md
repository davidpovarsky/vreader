sidebar** leg that round-4 (`feature-34-20260509.md`) reported as

## Observations

- **The fix is exactly the shape bug #155 sketched.** Round-4's
  bug body proposed: "derive `displayedBooks` in `LibraryView` (or
  expose via `LibraryViewModel`) as `viewModel.books.filter {
  activeFilter.matches($0) }`". PR #454 implemented that — the grid
  now consults `activeFilter` before rendering, the bookCount in the
  sidebar (always-correct per round-4) and the rendered set are now
  consistent.
- **Filter clears on All Books tap, not on sidebar dismiss.** The
  filter persists across sidebar re-opens — only an explicit
  `All Books` tap (or a different filter tap) clears it. This is
  the right UX for a persistent organize-by-collection workflow.
- **CU clipboard paste is still required for iOS Sim text fields**
  (same as round-3/round-4 observation; `mcp__computer-use__type`
  doesn't reach these fields). External CU/Sim infra quirk, not a
  bug.
- **Long-press timing is forgiving** — a 1-second mouse-down via
  `left_mouse_down` + `wait` + `left_mouse_up` reliably triggers
  the context menu in this build. No flakiness observed.
- **Sidebar count badge visibly increments** from 0 to 1 when the
  first book is added (data-layer round-1 already covered this; the
  new evidence is the visual confirmation that the badge re-renders
  immediately, not on a refresh).
- **No new bugs surfaced.** This is a clean round-5 close-the-loop
  for one of the deferred legs from round-4.
