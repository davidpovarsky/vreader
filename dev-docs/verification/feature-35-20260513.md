# Feature #35 — Annotations export/import UI surface verification

Closes feature #35's UI-driven slice via the WI-3 XCUITest harness
(landed in v3.21.6, PR #587, 2026-05-13 8:40 AM).

Round history:
- Round-1 (2026-05-07): 47-test data-layer slice (`AnnotationListViewModel`,
  exporter aggregator, JSON parser, Markdown formatter, importer
  round-trip). Result: partial — UI flow deferred.
- Round-2 (this iteration): `Feature35AnnotationsExportVerificationTests`
  (WI-3) drives the live UI surface end-to-end via XCUITest.

The two acceptance criteria checked here are the UI-trigger surfaces:
the Export button is visible + hittable, and the Import button is
visible + hittable. The button-press behaviors (UIActivityViewController
share sheet for Export, UIDocumentPicker for Import) are system-managed
modals that XCUITest cannot reliably observe from inside the app
process — these are explicitly out of scope per WI-3 plan note ("Can't
assert exported file contents via XCUI; assert sheet presence only").
