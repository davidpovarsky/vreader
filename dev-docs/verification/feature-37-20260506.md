## Summary

First device-verification round for feature #37 (per-book reading settings). Status was DONE-but-unverified since A05 shipped. The verification surface for per-book overrides came together via three recent fixes:

- Bug #126 / PR #305+#308 — eval surface for EPUB body inspection.
- Bug #145 / PR #324 — bridge-driven theme observer respects per-book override semantics.
- Bug #147 / PR #325 — disabling per-book re-resolves from globals (`reconcileFromDefaults`).

This slice exercises the **OPEN-with-override** path (the primary use case for feature #37): write a per-book JSON file, open the EPUB, observe that the configured fields apply and unconfigured fields inherit from globals.

## Two scenarios verified

### Scenario A — full override (theme + fontSize + lineSpacing)

Per-book file: `{"themeName":"sepia","fontSize":26,"lineSpacing":1.8}`.

| Field | Expected | Observed | Result |
|---|---|---|---|
| body backgroundColor | sepia bg | `rgb(244, 237, 221)` | pass |
| body color | sepia text | `rgb(58, 43, 22)` | pass |
| body fontSize | 26 px | `26px` | pass |
| `<p>` fontSize | 26 px | `26px` | pass |
| `<p>` line-height | 26 × 1.8 = 46.8 px | `46.799999px` | pass |

### Scenario B — partial override (fontSize only, theme inherits global dark)

Per-book file: `{"fontSize":30}`. Global UserDefaults: `readerTheme = "dark"`.

| Field | Expected | Observed | Result |
|---|---|---|---|
| body backgroundColor | global dark | `rgb(28, 28, 30)` | pass |
| body fontSize | 30 px (per-book) | `30px` | pass |

## Commands run

```bash
DATA=$(xcrun simctl get_app_container booted com.vreader.app data | tail -1)
PB_DIR="$DATA/Library/Application Support/PerBookSettings"
KEY="epub:f284fd...:2198"
SANITIZED="${KEY//:/_}"

# Scenario A — full override
xcrun simctl openurl booted "vreader-debug://reset"
xcrun simctl openurl booted "vreader-debug://seed?fixture=mini-epub3"
mkdir -p "$PB_DIR"
echo '{"themeName":"sepia","fontSize":26,"lineSpacing":1.8}' > "$PB_DIR/${SANITIZED}.json"
xcrun simctl openurl booted "vreader-debug://open?bookId=${KEY}"
# Probes: body-bg, body-color, body-fontsize, p-fontsize, p-lineheight

# Scenario B — partial override (fontSize only)
xcrun simctl terminate booted com.vreader.app
echo '{"fontSize":30}' > "$PB_DIR/${SANITIZED}.json"
xcrun simctl launch booted com.vreader.app
xcrun simctl openurl booted "vreader-debug://open?bookId=${KEY}"
# Probes: body-bg (expect dark from global), body-fontsize (expect 30 from per-book)
```

## Observations

- **Eval surface unblocked the verification** — pre-bug-#126, only positions/highlightCount were observable from the EPUB renderer. Per-book typography + theme application is now real-render-correlated via `getComputedStyle`.
- **Computed line-height precision**: 46.799999px vs the expected 46.8px is a CSS rounding artifact, not a bug. The implementation passes the lineSpacing multiplier to CSS; the WebView's float computation produces the trailing 9s.
- **Partial override correctly inherits**: body bg respects global dark (`rgb(28, 28, 30)`) when per-book leaves `themeName` nil, while fontSize is overridden to 30. Matches the documented "nil = inherit from global" contract.
