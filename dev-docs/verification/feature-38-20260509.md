## Observations

- The mini-epub3 TOC sheet renders cleanly: "Chapter One" highlighted (current chapter) and "Chapter Two" both at the same x-offset and same font weight. This confirms the level=0 rendering branch but doesn't exercise level≥1.
- The war-and-peace.txt and mini-azw3 TOC sheets show the empty state ("No table of contents available for this document"). For war-and-peace this is consistent with bug #152 / feature #23 round-2's finding that Project Gutenberg-style chapter headings ("CHAPTER I", "CHAPTER II", …) don't trigger any of the 25 Legado regex rules in `TXTService`. For mini-azw3 ("Masque of the Red Death", a single-section short story), Foliate-js correctly reports an empty TOC — the source MOBI has no `<guide>` or NCX entries.
- **Not a defect.** The implementation is correct (per round-1's 69-test suite + code-read). What's missing is *test infrastructure*: a bundled fixture with deliberately-nested TOC entries for both EPUB (NCX with multiple `<navPoint>` levels) and PDF (PDFOutline with sub-outlines). Without one, every future verify-cron firing will hit the same fixture block.
- **Recommended follow-up** (out of scope for this iteration): add a `mini-nested-toc.epub` fixture with at least 3 levels of `<navPoint>` nesting (e.g., Part → Chapter → Section). 5-10KB EPUB is enough. Then feature #38's UI-render leg flips from PARTIAL to PASS.
- No bugs observed. The user-facing behaviour for the cases that *are* exercisable (single-level TOC + empty TOC) is correct.
