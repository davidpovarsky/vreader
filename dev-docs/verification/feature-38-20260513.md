# Feature #38 — Hierarchical/tree TOC display (round-4 — visual hierarchy slice)

Closes the **visual hierarchy** slice that round-2 and round-3 left open
("level ≥ 1 indent + font differentiation needs a nested-TOC fixture").
The Test Markdown TOC seed exposes 3 distinct heading levels (H1 / H2 /
H3) — exactly the input shape the deferred slice needed.

Tap-to-navigate slice already closed by round-3 (`feature-38-20260510-round3.md`).

## Acceptance criteria

| Criterion | Observed | Pass/Fail |
|---|---|---|
| TOC extracted from MD headings | TOC panel lists 5 entries matching the 5 ATX headings: Introduction, Chapter 1: The Beginning, Chapter 2: The Middle, Section 2.1: A Subsection, Chapter 3: The End. | pass |
| Level-based indentation rendered | 3 distinct indentation tiers visible in panel: Introduction at left margin (0pt), Chapter rows indented ~20pt, Section 2.1 indented ~40pt (more than Chapter rows). Matches `TOCRowView.indentPerLevel = 20pt × entry.level`. | pass |
| Multi-level nesting (level ≥ 2) | "Section 2.1: A Subsection" (level 2, H3) renders at the deepest tier, visually nested under "Chapter 2: The Middle" (level 1, H2). Three nesting depths exercised, not just two. | pass |
| Font differentiation between levels | Introduction (level 0) renders in body font + medium weight. Chapter + Section rows (level 1+) render in subheadline font + regular weight, secondary color. Per `TOCRowView.swift:125-127`. | pass |
| Active-entry visual cue | Introduction is the current section (reader at scroll position 0%) and renders in accent (blue) color + semibold weight, distinct from inactive entries. | pass |
| Tap-to-navigate (carried over) | Round-3 closed this slice on mini-epub3 (Chapter Two row tap → reader navigated Chapter 1 of 2 → Chapter 2 of 2, panel auto-dismissed, bottom chrome flipped). | pass (round-3) |

## Observations

- The seed text uses `# Introduction` (single `#`, H1) → `TOCBuilder.forMD`
  computes `level = hashCount - 1 = 0`. `##` Chapter rows → level 1.
  `### Section 2.1` → level 2. Aligns with the documented contract at
  `TOCBuilder.swift:106-107` ("Returns entries in document order with
  level = (hash count - 1)").
- The 20pt-per-level indentation produces visually unambiguous tiers
  even at the small iOS sheet width. Section 2.1's 40pt indent is
  visibly deeper than the 20pt Chapter rows in the screenshot.
- Active-entry highlighting (accent color + semibold) is independent
  of level-based color/weight differentiation. The active state wins
  precedence in the conditional at `TOCRowView.swift:126-127`.
- This is the first verification round to exercise 3 distinct level
  values (0, 1, 2) end-to-end in the UI. Rounds 1-2 had only flat
  TOCs from bundled fixtures; round 3 exercised level 0 only on
  `mini-epub3` (Chapter One / Chapter Two — same level).
