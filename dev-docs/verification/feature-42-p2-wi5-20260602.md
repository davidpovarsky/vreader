# Feature #42 Phase 2 (Kindle convert-on-import) — WI-5 device verification

Verifies the convert-on-import path (WI-1a..4b) end-to-end on device, behind the
`--enable-kindle-convert` DEBUG launch flag (the feature ships dark — default
OFF — until a separate human-gated flip).

## Acceptance criteria

| # | Criterion | Observed | Result |
|---|---|---|---|
| 1 | Convert-on-import runs for a Kindle file when the flag is ON | Log: `[BookImporter] Kindle convert-on-import: converted source to EPUB` | ✅ |
| 2 | The Kindle book imports as a first-class EPUB (not azw3) | Persisted row: `format=epub`, `fingerprintKey=epub:270b…:10377479` (the converted blob's own identity — self-consistent) | ✅ |
| 3 | Display metadata is recovered from the self-describing EPUB | Title `被讨厌的勇气：…` + author `（日）岸见一郎，（日）古贺史健 著；渠海霞 译` — the book's REAL embedded metadata, not the filename | ✅ |
| 4 | Deterministic dedup (re-import the same AZW3 → same identity) | Re-import logged `(duplicate=true)` | ✅ |
| 5 | Flag gating (OFF → native azw3) | A prior flag-OFF import persists as a distinct `azw3:39826…` row alongside the converted `epub:270b…` row | ✅ |
| 6 | The converted EPUB RENDERS via the EPUB engine | Opened the converted book → reader displays its CJK content (copyright page: 书名/作者/出版社/ISBN) — see artifact | ✅ |

## Commands run

```bash
# Build + install the Debug build (v3.42.24), boot iPhone 17 Pro sim.
xcrun simctl install <udid> vreader.app
# Force the flag + import the real 6 MB CJK AZW3 from test-books/.
xcrun simctl launch <udid> com.vreader.app --enable-kindle-convert
xcrun simctl openurl <udid> "file:///tmp/wi5-kindle.azw3"
# Inspect the persisted store + open the converted book.
sqlite3 default.store "SELECT ZTITLE, ZFORMAT, ZFINGERPRINTKEY FROM ZBOOK;"
xcrun simctl openurl <udid> "vreader-debug://open?bookId=epub:270b…"
```

## Observations

- The convert path produces a **self-consistent** EPUB: the persisted
  `fingerprintKey` (`epub:<sha256-of-epub>:<epub-byteCount>`) describes the stored
  blob, so backup/restore/dedup all operate on a genuine EPUB (the 4-round Gate-2
  design's central invariant, confirmed live).
- **Known gap (non-load-bearing):** the two best-effort `ImportProvenance`
  Kindle-origin fields (`convertedFromKindleExtension`, `converterVersion`) are
  set on the in-memory `ImportResult` (unit-tested) but NOT persisted onto the
  `Book` row — the `BookRecord`→`Book` mapping doesn't thread them. Per design
  decision #5 this is observability-only and its loss is acceptable, so it does
  NOT affect correctness/rendering/identity. Filed as a follow-up to complete the
  persistence mapping.
