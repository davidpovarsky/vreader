## Observations

- **The card formerly known as "AZW3 cover gap"**. Round-1 (2026-05-05)
  reported AZW3 cover not rendering for "被讨厌的勇气-1"; round-2
  (synthetic EPUB) and round-3 (live mini-azw3 fixture) progressively
  closed gaps; round-4 confirms the UI grid render leg, which was the
  last deferred leg. Three independent failure modes were resolved
  along the way: bug #122 (EPUB redundant-prefix href, FIXED), the
  AZW3 device-verification fixture gap (resolved by bundling
  mini-azw3 in DEBUG builds), and bug #340 (EXTH title FIXED + closed).
- **Cover JPEG round-trip is perceptibly correct**. The 341×512 JPEG
  stored on disk visually matches the library card byte-for-byte
  (cyan background, white headline "The Masque of the Red Death",
  "Edgar Allan Poe" subhead, "Project Gutenberg" footer, abstract red
  geometric figure). No banding, no aspect distortion, no white-edge
  padding (bug #107 is a separate row).
- **Synthetic fallback for cover-less EPUBs is the correct behavior**.
  mini-epub3.epub contains no `OEBPS/Images/cover.jpg` or equivalent
  per `unzip -l`. The library renders a synthetic blue cover with
  "EPUB" label and white book glyph, which is the documented
  fallback path. This is NOT a feature-43 regression — feature #43
  scope is "extract and display *when available*", not "always
  generate a cover".
- **No new bugs surfaced.** This is a clean round-4 close-the-loop
  verification. The feature is feature-complete for both EPUB and
  AZW3/MOBI scope.
