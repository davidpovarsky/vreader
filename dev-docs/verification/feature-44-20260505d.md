## Why partial, not pass

Plan-level criteria 4, 5, 7 are not satisfied by round 4's evidence:
- **Foliate `settle`/`eval` happy path** (criterion 4 + the eval portion of criterion 1's "all 7 commands"): TXT-only fixture means Foliate-specific behavior on EPUB is not tested. Adding an EPUB fixture to `DebugFixtureCatalog` (a 2-line catalog change + dropping a public-domain EPUB into `vreader/Resources/DebugFixtures/`) would close this for a future round 5.
- **All tests pass** (criterion 5): 19 pre-existing failures, all orthogonal to DebugBridge. A tracker-cleanup pass on those is a separate concern.
- **LOC <300** (criterion 7): `RealDebugBridgeContext.swift` is 491 lines. A file split (e.g., separate file per command handler, or split out the snapshot-payload builder) is a structural refactor — explicitly deferred per Codex's recommendation in the bug #124 audit.
- **Release-build absence-gate** (criterion 2): the script exists but wasn't run this round. Trivial to add.

The 7 URL handlers + 5 observable surfaces (logs, ZBOOK count, ImportedBooks files, sentinel JSON files, snapshot JSON) are all working as documented. Implementation is functionally complete — that's why the row moves to DONE — but the plan's strict acceptance gate is not yet met, which is why the result is `partial` and the row stays at DONE rather than VERIFIED.

## Observations

- **Linear validation chain**: each prior round identified one or two specific blockers. Round 4 confirms the chain is closed: register → approve → dispatch → seed → open → settle → snapshot → eval. No new bugs found.
- **Settle's two payloads validated**: the no-reader path (criterion 6) and the with-reader path (criterion 8) produce different documented JSON shapes, both observable, both correctly differentiated by the presence/absence of the `error` key and probe-shaped fields. Bug #125's helper extraction (`writeReadySentinel`) cleanly handled both.
- **Eval's error path validated**: TXT reader correctly reports `unsupported`, with the payload carrying `bridge/format/error/fingerprintKey/ts`. Eval's *happy* path on Foliate (EPUB) is NOT exercised this round because the DebugFixtureCatalog only contains `war-and-peace.txt`. Adding an EPUB fixture is trivial (drop a public-domain EPUB into `vreader/Resources/DebugFixtures/` and register a catalog entry) and would let a future round close the eval-happy-path criterion.
- **Snapshot lastError null at end of run**: even though commands like settle (no reader) and eval (TXT) "fail" from the harness's perspective, they don't propagate to `lastError` because both write sentinel files instead of throwing. Only the parse-failure paths set `lastError` now. This is the documented contract post bug #125.
- **Pre-existing test failure** (`test_snapshot_withoutActiveReader_listsReaderFieldsAsPartial` asserts `schemaVersion == 1`, actual 2) is unchanged and unrelated. The actual snapshot in this run reports `schemaVersion: 2`.
- All 7 commands respect the documented URL grammar — no parsing glitches encountered.
