## Observations

- The **content-addressing pipeline is verifiable** — fixture's SHA-256 prefix (`f284fd07...`) and byte count (`2198`) round-trip from fixture file → ImportedBooks blob path → ZBOOK fingerprintKey. This is a strong end-to-end check that EPUB ingestion stayed correct after the SchemaV4 migration earlier in the project.
- **Title extraction** worked on the synthetic fixture: `<dc:title>VReader Mini EPUB Fixture</dc:title>` from `OEBPS/content.opf` was visible in `ZBOOK.ZTITLE`. Confirms `EPUBMetadataExtractor` happy path on EPUB 3 reflowable fixtures.
- **Snapshot deltas are honest**: `partial: ["selection", "currentBookId", "format", "position"]` after seed (no book open); shrinks to `partial: ["selection", "position"]` after open (currentBookId + format known, position is null because the EPUB freshly loaded — but `position` would only be non-null if a positionProvider was wired, which it isn't because of bug #126).
- **Round 4d's framing of the gap was incomplete.** "Needs an EPUB fixture" was true but not sufficient — the fixture lets us drive the surface, but criterion (d) requires the WKWebView's render-complete signal to actually gate `awaitSettle`. The probe adapter has the right hook shape (`settleStrategy` closure) but no reader host plugs it in. Bug #126 is the remaining fix.
- **Eval's "unsupported" error is correct as documented**, but the doc itself reveals the gap: `DebugReaderProbeAdapter.swift:12-21` literally says _"no reader supplies a live evaluator yet, so eval?bridge=foliate against real EPUB/AZW3 books always writes evalUnsupported"_. So `evalUnsupported` is current expected behavior, not a regression — but criterion (d) needs that to change.
- **Iteration cost is low** once the harness is fixed in place. End-to-end seed → open → settle → snapshot → eval against the EPUB fixture is ~12 seconds wall-clock + simctl install. Future verification rounds against EPUB will be cheap.
