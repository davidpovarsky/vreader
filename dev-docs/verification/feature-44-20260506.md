## Observations

- **iOS 26.4 portability now solid**: rounds 4d, 5, 6 ran on earlier iOS 26 builds where the legacy plist-only grant worked. PR #302 makes the grant script work on both old (legacy plist honored) and new (SQLite-only) iOS Simulator images.
- **First end-to-end DebugBridge run with screenshot evidence**: prior rounds had headless JSON evidence; round 7 captures actual rendered-reader screenshots showing both formats opening correctly. Artifacts committed under `dev-docs/verification/artifacts/`.
- **EPUB snapshot oddity**: the screenshot of the EPUB-open shows the reader scrolled to "Chapter Two" content (end of fixture) rather than the start. Likely an auto-restore-position artifact from the prior session's open of mini-epub3 â€” not a regression. The snapshot JSON has `partial: ["selection", "position"]` so the position is null; the rendered scroll position is preserved from the previous session in the per-book ReadingPosition row.
- **Bug #126 still the hard blocker** for criterion (d): probe adapter's `awaitSettle` returns from a 100ms placeholder; `evaluateJavaScript` still produces `evalUnsupported` for EPUB. Settling and eval at the COMMAND level work end-to-end (sentinel files written, error JSON shape correct), but the FORMAT-LEVEL behavior promised by criterion (d) â€” actually waiting for Foliate `relocate` and actually evaluating JS in the EPUB WebView â€” remains unimplemented.
