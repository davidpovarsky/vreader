## Summary

Round-12 quick-smoke verification of feature #44 (DebugBridge) on
merged-main `41067e5` (v3.14.123, build 232). All 7 URL handlers
exercised end-to-end against the merged build to validate no regression
since round-11 (2026-05-09 against v3.14.121, commit `a458a69`).

**Result: PASS.** All 7 handlers behave per spec. The eval handler's
"unsupported for format: epub" return when called against an EPUB in
Unified mode is correct architecture (Unified uses TextKit reflow, not
WKWebView — no JS context to evaluate against). After switching to
Native mode (which uses `EPUBWebViewBridgeCoordinator` + WKWebView),
eval returns real JS results identical to round-10's reference
(`docTitle: "Chapter One"`, `paragraphCount: 3`, `readyState: "complete"`).

This round confirms the recent merges (PR #483-486 around bug #162,
PR #481 around bug #161 / #478, PR #492 around bug #166 logging) did
not regress the DebugBridge surface.

## Commands run

```bash
SIM_ID=FDF2EA2A-532E-48D4-9022-ADEB6CD053CC  # iPhone 17, iOS 26.4

# Handlers 1-3: reset → seed × 3 → theme
xcrun simctl openurl booted "vreader-debug://reset"
xcrun simctl openurl booted "vreader-debug://seed?fixture=war-and-peace"
xcrun simctl openurl booted "vreader-debug://seed?fixture=mini-epub3"
xcrun simctl openurl booted "vreader-debug://seed?fixture=mini-azw3"
xcrun simctl openurl booted "vreader-debug://theme?mode=dark&fontSize=18"

# Handler 6: snapshot post-seed
xcrun simctl openurl booted "vreader-debug://snapshot?dest=lib_snapshot.json"

# Inspect SwiftData store directly:

sqlite3 "$STORE" "SELECT ZFINGERPRINTKEY, ZTITLE FROM ZBOOK;"
# txt:ab0252...:1708|war-and-peace
# epub:f284fd...:2198|VReader Mini EPUB Fixture
# azw3:fadbaa...:128650|The Masque of the Red Death

# Handlers 4-5: open + settle
xcrun simctl openurl booted "vreader-debug://open?bookId=epub:f284fd...:2198"
xcrun simctl openurl booted "vreader-debug://settle?token=t1"
xcrun simctl openurl booted "vreader-debug://snapshot?dest=after_open.json"

# Handler 7: eval (Unified mode — returns "unsupported"; Native mode — returns real result)
JS_B64=$(echo -n "JSON.stringify({ docTitle: document.title, paragraphCount: document.querySelectorAll('p').length, readyState: document.readyState })" | base64)
xcrun simctl openurl booted "vreader-debug://eval?bridge=epub&js=$JS_B64"

# Switched to Native via UI: AA settings → tap Native segment at AX (544, 882)
xcrun simctl openurl booted "vreader-debug://eval?bridge=epub&js=$JS_B64"
# → returns real DOM-state JSON
```

## Observations

- **All 7 handlers behave per documentation**. Round-12 is a clean
  smoke pass; no regressions from PR #483-486 (bug #162 work),
  PR #481 (bug #161/#478), or PR #492 (bug #166 logging).
- **Eval handler behavior depends on reading mode** (correct
  architecture):
  - Unified mode: TextKit-based reflow, no WKWebView → eval returns
    `{ "error": "eval unsupported for format: epub" }` with correct
    fingerprintKey + format fields.
  - Native mode: `EPUBWebViewBridgeCoordinator` registers its
    `WKWebView` to `DebugReaderRegistry.activeEPUBWebView` from the
    `webView(_:didFinish:)` hook (PR #305 / v3.14.13 wiring). Eval
    against this registry returns real JS results.
- **Snapshot schema v2 stable**: all documented fields present,
  `partial` array correctly listing nil-valued state (selection +
  position when no selection / position is set).
- **theme command propagation works**: `theme?mode=dark&fontSize=18`
  reflected in snapshot and visible in reader UI (dark background,
  18pt font). This was the bug #144 fix (notification-driven
  propagation to live `ReaderSettingsStore`); confirmed still
  working.
- **Container path stable**: app data container at standard
  `Library/Caches/DebugBridge/` location; both snapshot and eval
  result files write reliably.
- **No bugs filed.** Surface is clean.
