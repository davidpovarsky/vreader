# Feature #44 — DebugBridge round-15 guardrail (v3.21.3)

Round-15 smoke pass after feature #45 WI-1 (verification harness scaffold,
PR #581, v3.21.3) landed. This is 30 versions newer than round-14 (v3.16.1),
spanning PRs #542–#582. The DebugBridge handlers ARE the verification plumbing
for the autonomous verify cron, so any regression here would silently break
all future verify-cron iterations.

All 7 documented handlers verified end-to-end on v3.21.3 build 280
(merge commit `6ad6193682ac763a134cc85d490be4c6d18b726c`, tag `v3.21.3`)
against iPhone 17 Pro Simulator (iOS 26.5).

## Commands run

```bash
# Reset library state
xcrun simctl openurl booted "vreader-debug://reset"

# Seed bundled EPUB fixture
xcrun simctl openurl booted "vreader-debug://seed?fixture=mini-epub3"

# Set theme to dark
xcrun simctl openurl booted "vreader-debug://theme?mode=dark"

# Snapshot baseline state (library-level)
xcrun simctl openurl booted "vreader-debug://snapshot?dest=verify-r15-baseline.json"

# Look up bookId from SwiftData store
CONTAINER=$(xcrun simctl get_app_container booted com.vreader.app data)
FP=$(sqlite3 "$CONTAINER/Library/Application Support/default.store" \
  "SELECT ZFINGERPRINTKEY FROM ZBOOK WHERE ZFORMAT='epub' OR ZFORMAT1='epub' LIMIT 1;")
URL_ENC=$(python3 -c "import urllib.parse,sys; print(urllib.parse.quote(sys.argv[1], safe=''))" "$FP")

# Open the EPUB
xcrun simctl openurl booted "vreader-debug://open?bookId=$URL_ENC"

# Wait for reader to settle
xcrun simctl openurl booted "vreader-debug://settle?token=verify-r15-settle"

# Eval JS in the live EPUB WKWebView (bug #171 cross-check)
JS='(function(){var b=document.body;return JSON.stringify({title:document.title,scrollWidth:b.scrollWidth,clientWidth:b.clientWidth,columnCount:getComputedStyle(b).columnCount,bodyTextLen:(b.innerText||"").length});})()'
JS_B64=$(printf '%s' "$JS" | base64)
xcrun simctl openurl booted "vreader-debug://eval?bridge=epub&js=$JS_B64"
```

## Observations

- All 7 handlers pass. No regressions across the 30 versions (PRs #542–#582)
  since round-14.
- **`columnCount` change**: round-14 (iOS 26.4) reported `"1"`, round-15
  (iOS 26.5) reports `"auto"`. The observable behavior is unchanged:
  `scrollWidth === clientWidth === 980` confirms single-column paged mode.
  The discrepancy is a WebKit computed-style reporting difference between
  iOS versions — when `column-count: 1 !important` is in effect, iOS 26.5's
  WebKit resolves `getComputedStyle().columnCount` to `"auto"` (treating
  single-column as the natural/auto state). Bug #171 (FIXED in PR #541)
  remains effective; no regression.
- The snapshot is 424 B (vs 463 B in round-14) — the compact size change is
  expected: fields such as `position` are null/empty on a fresh-seeded library,
  which serializes more compactly than a mid-read state.
- The settle sentinel is 209 B — identical to round-14, stable.
- The eval result is 305 B — slightly larger than round-14 due to rounding in
  the `title` field; functionally identical.
- Feature #45 WI-1's new `VerificationDebugBridgeHelper` is tested separately
  via the Swift Testing suite; this round confirms the underlying handler
  contract it wraps is still correct.
