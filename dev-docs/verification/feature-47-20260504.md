## Observations

- **The verification path itself was the most informative work.** It caught a silent regression (bug #B2 — RemoteBookCatalog ISO8601 date decode) that 100% of unit tests passed because the unit fixtures used the default Double Date encoding while real backups use ISO8601 strings. The fix landed in commit 3f6dd55. This is exactly the scenario the new PreToolUse hook framework is meant to enforce: shipping #47 to DONE without verification would have meant a broken picker for every real user.
- **Bug #B1 (rclone tmp dir 409)** surfaces only on the first ever backup with this rclone setup. Subsequent backups succeed. Worth a feature #46 follow-up that MKCOLs the tmp path before PUT, OR documents the rclone setup requirement.
- **Bug #B3 (BookFileMaterializer alreadyLocal silently skips SwiftData insert)** is a #46-era bug. Materializer assumes "file exists at canonical path" implies "SwiftData row exists for it." When the row was deleted but the file wasn't (because LibraryViewModel.deleteBook only nukes the row, not the sandbox file), restore reports success but the row never appears. Not a #47 regression but worth filing.
- **Bug #B4 (row tap doesn't trigger sheet)** is the most concerning. Could be: (a) click coordinates landing wrong because the Calendar window overlapped the lower simulator area, (b) Environment values nil-coalescing (`lazyDownloadCoordinator` / `webDAVNetworkPolicy` not propagating from VReaderApp through ContentView → LibraryView), (c) the LibraryView observer not registered in time, (d) some other wiring gap. Needs dedicated investigation in a fresh session with a cleaner desktop layout.
- **Hardware keyboard typing was unreliable** in the simulator — iOS predictive text intercepted nearly every "type" call. Workaround was clipboard write + Cmd+V. Documenting for future verification runs.
