## Observations

**iOS 26 accessibility flattening required a workaround**: In iOS 26 SwiftUI, `.accessibilityIdentifier` on a container ZStack propagates to the inner UITextView, overriding inner view identifiers. `txtChapterTitleOverlay` (a `Text` inside the ZStack) is not independently accessible via `XCUIElement`. Fixed by encoding chapter mode state into `.accessibilityValue` of the container element (`"restoredOffset:N chapterMode:true chapters:M"`), matching the existing pattern used by bug #160 tests for `restoredOffset:N`.

**AnnotationsPanelView has no close button**: The sheet is dismissed only via swipe gesture or `onDismiss()` closure. The persistence test was refactored to use `app.terminate()` + relaunch with `keepExisting` seed instead of navigating back through the reader. This is actually the stronger persistence assertion — it tests that data survived a full process termination (disk-backed SwiftData store), not just an in-session navigation.

**`makeLocator` logging flood during reader navigation**: The TXT chapter-mode reader continuously fires `makeLocator` calls while open (appears to be triggered by the auto-scroll or reading position tracking). Not a regression — was present before Feature #48.

**TXTChapterModeHighlightLocator**: The global↔chapter-local offset translation in `TXTChapterModeHighlightLocator` correctly handles the war-and-peace.txt fixture (4 chapters: preamble + Chapters 1–3). Long-press on the first chapter's content produces a locator with `href=txtchapter:0:0` that survives the terminate+relaunch cycle.
