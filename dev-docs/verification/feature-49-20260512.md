## Commands run

Reproducible recipe. The simulator should be booted before starting.

```bash
# Capture booted simulator UDID
SIM_ID=$(xcrun simctl list devices booted | awk '/Booted/ {print $(NF-1); exit}' | tr -d '()')

# Install fresh Debug build
APP=/Users/ll/Library/Developer/Xcode/DerivedData/vreader-hdhlhcqmxppsadhececcxeadpkvz/Build/Products/Debug-iphonesimulator/vreader.app
xcrun simctl install $SIM_ID "$APP"

# Pre-grant the DebugBridge URL scheme approval
bash scripts/grant-debug-scheme-approval.sh $SIM_ID

# Reset library, seed mini-epub3, set light/18px theme
xcrun simctl openurl $SIM_ID "vreader-debug://reset"
xcrun simctl openurl $SIM_ID "vreader-debug://seed?fixture=mini-epub3"
xcrun simctl openurl $SIM_ID "vreader-debug://theme?mode=light&fontSize=18"

# Open the seeded book
BOOK_KEY="epub:f284fd074ccd1d3c1a78985464d9e1be27975f4029f3c2ddef8428ca10684af4:2198"
ENCODED=$(printf '%s' "$BOOK_KEY" | sed 's/:/%3A/g')
xcrun simctl openurl $SIM_ID "vreader-debug://open?bookId=$ENCODED"
xcrun simctl openurl $SIM_ID "vreader-debug://settle?token=open"

# PORTRAIT PROBE — paddingTop, paddingBottom, first element position
JS='JSON.stringify({paddingTop: getComputedStyle(document.body).paddingTop, paddingBottom: getComputedStyle(document.body).paddingBottom, firstElTop: document.body.firstElementChild ? document.body.firstElementChild.getBoundingClientRect().top : null, firstElTag: document.body.firstElementChild ? document.body.firstElementChild.tagName : null});'
B64=$(printf '%s' "$JS" | base64)
xcrun simctl openurl $SIM_ID "vreader-debug://eval?bridge=epub&js=$B64"

# Read the result
DATA=$(xcrun simctl get_app_container $SIM_ID com.vreader.app data)
cat "$DATA/Library/Caches/DebugBridge/eval-epub.json"

# Rotate to landscape via Simulator.app menu
osascript <<'EOF'
tell application "Simulator" to activate
tell application "System Events"
    tell process "Simulator"
        click menu item "Rotate Left" of menu "Device" of menu bar 1
    end tell
end tell
EOF

# LANDSCAPE PAGED-MODE PROBE — all invariants in one eval
JS='var b=document.body; var s=getComputedStyle(b); JSON.stringify({columnCount: s.columnCount, scrollWidth: b.scrollWidth, clientWidth: b.clientWidth, pageCount: Math.ceil(b.scrollWidth / b.clientWidth), paddingTop: s.paddingTop, paddingBottom: s.paddingBottom, bodyHeight: b.clientHeight, scrollLeft: b.scrollLeft, viewportWidth: window.innerWidth, viewportHeight: window.innerHeight, orientation: window.orientation});'
B64=$(printf '%s' "$JS" | base64)
xcrun simctl openurl $SIM_ID "vreader-debug://eval?bridge=epub&js=$B64"

# PAGE-ADVANCE PROBE — set both html and body scrollLeft (mirrors production EPUBPaginationHelper)
JS='var b=document.body; var h=document.documentElement; var t=b.scrollWidth - b.clientWidth; h.scrollLeft = t; b.scrollLeft = t; JSON.stringify({html: h.scrollLeft, body: b.scrollLeft, target: t, scrollWidth: b.scrollWidth, clientWidth: b.clientWidth});'
B64=$(printf '%s' "$JS" | base64)
xcrun simctl openurl $SIM_ID "vreader-debug://eval?bridge=epub&js=$B64"
```

## Observations

1. **2em math exact**: at fontSize=18px, 2em = 36px. The probe returned exactly 36px (not 32px or some rounded approximation), confirming WebKit honors the literal 2em declaration.
2. **First element top at 60.109px in portrait, not 36px** — the gap is the existing safe-area + DI cushion from bug #163 (`scrollView.contentInset.top`). The 2em body padding stacks on top of it: 36px body padding + ~24px safe-area inset = ~60px observed. This matches the plan's stated interaction with bug #163 ("Net: MORE breathing room — improves bug #163's fix rather than fighting it").
3. **Padding survives landscape rotation**: paddingTop/Bottom stayed at 36px when viewport collapsed from 2131px high to 525px high. No padding collapse from short-viewport conditions, as feared in round-1 audit M2.
4. **bodyHeight 946px in landscape with viewportHeight 525px**: body's `clientHeight` exceeds viewport. This is expected for the EPUB renderer — the body is laid out at content height; the WebView's scroll container handles overflow. Important is that body height is POSITIVE (not collapsed to zero by aggressive padding).
5. **mini-epub3 is single-page**: pagination math worked but couldn't be stressed across multiple pages (scrollWidth = clientWidth = 980, single page). Page-advance probe set scrollLeft to `target = 0` (the only valid position for a single-page chapter), confirming html and body scrollLeft stayed in sync — no math break.
6. **Theme switch did not propagate to live WebView**: after firing `theme?mode=dark`, the snapshot shows `theme: "dark"` but the live WebView's computed `backgroundColor` stayed at `rgb(255,255,255)` (light theme colors). This is a separate issue — the open book doesn't re-inject CSS on theme change; it would on next book open. Padding verification is theme-independent (unit tests cover all three themes), so this didn't block Gate 5.
7. **CSS columns + body height**: didn't expose any breakage. The plan's concern about "last page clipping" or "fractional page count" didn't materialize in this fixture (single-page). Future verification with a multi-chapter EPUB fixture would strengthen this assertion, but is not blocking.
