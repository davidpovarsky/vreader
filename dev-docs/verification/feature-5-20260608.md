# Feature #5 — Search highlight auto-dismiss on next action (Gate-5b, round 4)

Closes the gap that held three prior rounds at `fail`/`partial`: the temporary
yellow search highlight could never be visually confirmed in the viewport
(2026-05-04 / -05-08 off-viewport; 2026-05-13 `fail` — no paint, attributed to
bug #154 repeat-nav no-op + a CU screenshot-timing artifact). Both prerequisite
bugs (#153 scroll-anchor, #154 repeat-nav no-op) are now FIXED+closed on main,
and `simctl io screenshot` provides the visual assertion the prior CU
virtual-display path could not.

## Observations

- The DebugBridge search driver (`search?query=&index=`, Bug #238) returns the
  correct single match for the deterministic `Paragraph 50` term — the
  in-viewport landing the prior off-viewport `Paragraph 100` term lacked.
- The `&index=N` result-tap raced ahead of results being ready (no nav); the
  two-step open-sheet → idb-tap-result path is reliable. The search sheet's
  detent height varies between presentations, so the result-row tap point must
  be read per-presentation, not fixed.
- `snapshot.position` reads stale (0) right after a navigate (it updates on
  scroll-settle, not on navigate) — the screenshot is the authoritative
  position+highlight signal, not the snapshot.
- Incidental: the drop-cap "P" on the document title is visible (feature #68).
