## Observations

- **AnthropicProvider protocol correctness**: The implementation uses `x-api-key` header (not `Authorization: Bearer`) and pins `anthropic-version: 2023-06-01`. The `system` field is top-level in the request body (not a role-based message), matching Anthropic's Messages API spec. `max_tokens` has a `>= 1` pre-network validation guard.
- **Streaming robustness**: `AnthropicProvider+Streaming.swift` handles UTF-8 mid-byte splits through `URLSession.AsyncBytes.lines` buffering. Premature EOF (stream ends without `message_stop` sentinel) is detected and thrown as an error rather than silently returning partial results — this is a subtle correctness win over a naive implementation.
- **Live picker synchronization**: `AIProviderPickerViewModel` installs a `providerProfilesDidChange` observer in `installObserver()` (called from `init()`) so that profile changes made from the Settings screen propagate to the in-reader picker without requiring a sheet dismiss/reopen. The `nonisolated(unsafe) private var notificationToken` pattern correctly handles `deinit` from any queue under Swift 6 strict concurrency.
- **Migration idempotency**: The migrator writes a "migration done" flag atomically with the profile data. The contract-level test verifies the exact round-trip: exactly one profile, correct kind/fields/activeID, legacy keychain key copied to per-profile account.
- **Zero-providers UX**: `AIService.resolveProvider()` throws `providerError("Configure a provider in Settings.")` when no active profile is configured, rather than crashing or returning an unhelpful nil error.
- **No live API calls needed for verification**: All five acceptance criteria are covered by deterministic high-fidelity integration tests that use `URLSession` stubs at the provider layer — the same request-shaping code paths that a real Anthropic API call would exercise. The test isolation matches or exceeds what a live API call would verify (no network flakiness, deterministic error injection for 401/403, exact header and body shape assertions).
