# Feature #52 — Multiple WebDAV server profiles with active-server switching

WI-6 round-1 verification of the multi-profile WebDAV feature on the merged-main
build that includes WI-1 through WI-5. Verifies UI shape and editor affordances
end-to-end; defers the full backup+restore-with-active-profile roundtrip to a
follow-up round because computer-use's `type` action can't reliably reach iOS
Simulator text fields in this environment (a CU/Simulator integration
limitation, not a vreader defect).

## Acceptance criteria

The feature row's contract:

| Criterion | Status | Notes |
|---|---|---|
| (a) user can add two WebDAV profiles and switch the active one — only the active server is used for backup | **partial** | UI shape verified: add/edit sheets render with all sections; existing profile is active (radio filled). Switch-active + per-profile backup observation deferred. |
| (b) existing single-server users migrate without re-entering credentials | **partial** | Migrator code-path covered by 6 `WebDAVProfileMigrator` unit tests at WI-2; live migration not exercised this round (legacy flat-keychain state pre-WI-5 is no longer reachable from the post-WI-5 build to set up cleanly). |
| (c) deleting the active profile falls back to remaining or disables backup | **deferred** | Swipe-delete confirmation flow verified at WI-4a Gate 5 slice with screenshots; live fallback observation against backup section deferred. |
| (d) backup + restore round-trip works with the selected profile | **deferred** | Requires text-input into Add Server sheet to point at test rclone server. CU `type` action not reaching iOS Simulator text fields this session. |

## Commands run

```bash
# 1. Confirm live rclone WebDAV is up on 8081 (user's existing setup)
pgrep -fl "rclone serve webdav"
curl -s -X PROPFIND -H "Depth: 0" http://localhost:8081/ -u vreader:<htpasswd-hash>
# -> running pid 79811, htpasswd-protected on port 8081

# 2. Spin up TWO test rclone WebDAV servers (don't touch the user's live data dir)
mkdir -p /tmp/vreader-test-webdav-a /tmp/vreader-test-webdav-b
rclone serve webdav /tmp/vreader-test-webdav-a --addr 127.0.0.1:8082 --user vra --pass passworda &
rclone serve webdav /tmp/vreader-test-webdav-b --addr 127.0.0.1:8083 --user vrb --pass passwordb &
# PROPFIND probe -> http 207 (auth OK) for both

# 3. Build + install fresh v3.21.61 (commit 942703b)
DEVELOPER_DIR=/Applications/Xcode.app/Contents/Developer xcodebuild build \
  -project vreader.xcodeproj -scheme vreader \
  -destination 'platform=iOS Simulator,name=iPhone 17 Pro'
xcrun simctl install booted "<path>/vreader.app"
xcrun simctl launch booted com.vreader.app

# 4. Reset library + seed mini-epub3
xcrun simctl openurl booted 'vreader-debug://reset?backups=true'
xcrun simctl openurl booted 'vreader-debug://seed?fixture=mini-epub3'

# 5. Drive Settings → WebDAV Backup → Servers → tap pencil → Edit sheet
#    Drive Settings → WebDAV Backup → Servers → tap + → Add sheet
#    (CU clicks worked fine; CU `type` did not reach iOS Simulator text fields)
```

## Observations

### What was verified (PASS)

**1. WI-5 cleanup is real.** The WebDAV Backup screen no longer contains the
legacy single-server form. After the v3.21.60 merge, the screen contains only:

- **Servers** section: one nav-link row titled "Servers" with subtitle
  "Manage saved WebDAV servers and switch the active one" + footnote
  "Add a WebDAV server above to enable backup."
- **Network policy** section: "Wi-Fi only for book downloads" toggle.

No Server URL / Username / Password text fields, no Test Connection or
Save Credentials / Remove Credentials buttons. The flat-keychain UX is
gone, confirmed end-to-end. Screenshot:
`feature-52-wi6-01-webdav-no-legacy-form-20260514.png`.

**2. WI-4a list view + active-row marker work.** Servers nav-link opens
"WebDAV Servers" list view with toolbar "+" (add) + back chevron. One
profile is present with filled radio + pencil-edit affordance + row title.
Screenshot: `feature-52-wi6-03-server-list-with-active-row-20260514.png`.

**3. WI-4b editor — Add mode** affordance divergence (bug #184 pattern):

- Title: "Add WebDAV Server"
- Toolbar: Cancel / Add (Add disabled while required fields empty)
- Sections: Name (placeholder `e.g. "Home Nextcloud" or "Work Synology"`),
  Endpoint (Server URL `https://` + Username `alice`), Password
  (`Enter Password` + footnote **"The password is stored only after the
  profile is saved. Enter the password above, then tap Add at the top of
  this sheet."**), Connection (footnote **"Save the profile first, then
  return here to test the connection."**)
- **Save Password / Test Connection buttons HIDDEN in add mode** — replaced
  with footnote text exactly as the bug #184 pattern specifies.

Screenshot: `feature-52-wi6-02-add-sheet-add-mode-affordances-20260514.png`.

**4. WI-4b editor — Edit mode** affordance divergence:

- Title: "Edit WebDAV Server"
- Toolbar: Cancel / Save (Save disabled until changes)
- Sections: Name (pre-populated), Endpoint (Server URL placeholder
  `https://dav.example.com/` + Username placeholder `alice`), Password
  (`Enter new password to update` + **`Save Password`** button visible),
  Connection (**`Test Connection`** button visible)
- HTTPS footnote: "HTTPS is recommended. HTTP is accepted for Tailscale
  and local-network WebDAV servers." — present in both modes (matches
  WI-4b plan).

Screenshot: `feature-52-wi6-04-edit-sheet-edit-mode-affordances-20260514.png`.

**5. Idempotency from prior state.** The profile from a prior session
persisted across `vreader-debug://reset?backups=true` — WebDAV profiles
are stored in UserDefaults under `com.vreader.webdav.profiles` and the
reset URL doesn't clear those keys (the reset is library-scoped). The
migrator's stable UUID `00000002-AAAA-...` should make this idempotent
across re-runs — observable behavior matches.

### What is deferred (DEFERRED, not failed)

- End-to-end "switch active → backup → only that server received the
  zip" requires typing creds + URL into the Add sheet. CU `type` action
  fired (each input attempt logged "Typed N grapheme(s)") but no
  characters appeared in any field. Hardware-keyboard toggle
  (`cmd+shift+k`) didn't help — pressing once dismissed the soft
  keyboard but typing still didn't land; pressing again re-showed the
  soft keyboard. This is a known CU/Simulator integration limitation
  (CU's keystroke layer doesn't always reach the simulator's text input
  pipeline), not a vreader defect.
- Test Connection live HTTP probe — same blocker.
- Delete-active-profile fallback behavior on the backup section —
  partially covered by WI-4a's screenshot evidence of the confirmation
  alert; live observation of backup-section disable deferred.
- Pre-WI-5 legacy single-server credential migration — set-up requires
  writing legacy flat-keychain entries via simulator, then upgrading the
  build. Not done this round.
