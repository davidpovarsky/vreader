# Feature #53 — Tap on highlighted text → inline edit/delete options — VERIFIED

Final Gate-5b acceptance. Closes the **only** remaining slice: criterion (c)
"tap existing highlight → edit/delete/color menu" on the **PDF** format. The
other four formats (TXT round-4, EPUB round-6, MD-scroll round-8, AZW3/Foliate
round-9) were already device-verified; the PDF slice was blocked solely on "no

slice is verified here.

## Observations

- The single-tap must land on a clean (no active text-selection) highlight; a tap
  while a selection is still active surfaces the iOS Copy/Select-All menu instead.
  Dismissing the selection first (one tap on non-highlighted text) then tapping the
  highlight reliably fires the #64 tap recognizer.
- Known cosmetic note (already documented on Bug #199's row, not a #53 defect): the
  Foliate inline menu anchors at the reader origin (`sourceRect == .zero`); the PDF
  popover anchors at the bottom sheet. Both surface the edit/delete affordances.
