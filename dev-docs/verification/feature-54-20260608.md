## The blocker

Criterion 3 is inside the feature's acceptance contract ("replacement rules work
in native EPUB **and** MD"), but the EPUB half was deferred to Phase D-1 and never
implemented. Per `SCHEMA.md`, a deferred in-contract criterion forces
`result: partial`, which cannot flip the row to `VERIFIED`.

**What changed since the prior rounds:** the original Phase D-1 *blocker*
(feature #42's engine decision + human ratification) is now **resolved** — #42 is
VERIFIED (2026-06-03) and EPUB renders by default via Readium, which provides the
Swift-side seam Phase D-1 needs. So criterion 3 is no longer dependency-blocked;
it is simply **un-built**.

## Disposition — needs a user scope decision (surfaced separately)

A verification run cannot resolve this — it re-produces `partial` every time. Two
paths, both outside verification scope:

- **(a) Implement Phase D-1** — wire `ReplacementRuleFetcher` → a Readium content
  transform/decoration for EPUB (CFI-safe), then run the eval-confirm acceptance

  flips to `VERIFIED`.
- **(b) Descope** — remove the native-EPUB-replacement-rules half from #54's
  acceptance criteria and re-file it as a standalone feature; #54 then flips to
  `VERIFIED` on criteria 1/2/4/5.

Recommendation: surface the choice to the user (done in the campaign summary).
