`result: partial`) device-verified criteria 1, 2, 3, 7 and left **criterion 4's

## Observations

- The key unblock was conceptual: XCUITest taps the `TTSControlBar` play/pause
  button by its state-driven accessibility label ("Pause" ⇄ "Resume"), so the
  `.speaking ⇄ .paused` transition is observable + drivable **without** CU. The
  AZW3 path reuses the format-agnostic `TTSService` + `TTSControlBar`, so the
  cycle behaves identically to TXT/EPUB/MD — feature #57 changed only the AZW3
  text source, which criteria 2/3 already confirmed feeds real speech.
- The AZW3 reader is the live Foliate WKWebView path (`FoliateBilingualContainerView`
  → `FoliateSpikeView`); the More-popover "Read aloud" row + top chrome resolve
  there (criterion 1), so `startReadAloud` reaches the control bar.
