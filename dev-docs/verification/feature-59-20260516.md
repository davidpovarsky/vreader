### Step B — simctl openurl for `.pdf` (criterion c — initial attempt)

```bash
xcrun simctl openurl booted "file:///tmp/feat59-fixtures/test.pdf"
```

**Result**: DB row count stayed at 3 — **PDF did NOT dispatch to vreader**.

for `simctl openurl` direct dispatch. To verify vreader receives `.pdf`,
must use the Share Sheet path explicitly (Step C below).

## Observations

- The Round-1 verifier correctly diagnosed `.markdown` as an Apple-side
  UTI gap; `net.daringfireball.markdown` does NOT carry `.markdown` in
  its `public.filename-extension` tag spec on iOS 26.5. Confirmed
  still-broken in round 2 (out of vreader's control). vreader handles
  `.md` correctly.
- The DEVICE-VERIFY ASYMMETRY between `simctl openurl` and Share Sheet:
  `simctl openurl` goes through LaunchServices and picks the Default
  handler ONLY. For documents where vreader is the Default (`.epub`,
  `.txt`, `.md`, `.azw3` / `.mobi` / `.prc` / `.azw` — no competing
  apps installed), `simctl openurl` is sufficient. For `.pdf` (where
  Preview is the Default), Share Sheet is the only path to test
  vreader's receive side. Round 1 missed this nuance and incorrectly
  marked `.pdf` as "deferred" rather than "method-unavailable" — round
  2 corrected by using Files-app + Share Sheet.
- The PDF preview in iOS Quick Look rendered the DeviceKit framebuffer
  mask file as mostly-black with a thin border. This is correct
  (framebuffer masks are designed as opaque overlays); the fixture
  choice was driven by file size (the only ≤10 KB PDFs on the host
  were these DeviceKit assets).
