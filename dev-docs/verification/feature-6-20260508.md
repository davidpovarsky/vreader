app restarts — Medium). Extends the 2026-05-07 evidence
(`feature-6-20260507.md`) which verified the **ViewModel layer** via

## Observations

- XCUITest works with the monitor physically off and with `CU display
  unavailable` from the MCP — the verify cron's hard dependency on
  Computer Use is unnecessary for any feature whose acceptance can be
  expressed as XCUIApplication taps. Worth filing the harness gap-fill
  as its own feature (~36 GH issues currently parked in
  `awaiting-device-verification` could be unblocked by it).
- `LibraryPopulatedTests` doesn't directly assert that the **selected**
  sort option (e.g. Title) is visually marked active in the menu.
  The acceptance criterion is "menu shows the option" not "selected
  state renders correctly" — that's a separate concern, not a regression
  from this evidence.
- The sort-picker-shows-all-options test is also de-facto coverage for
  feature #20 (Sort order reset/revert to default), since "Default"
  is one of the menu options the test would hit if added. #20's row
  already references its own `feature-20-20260507.md`; not updating
  here to keep this artifact scoped to #6.
