## Observations

- Storage rule from bug #151's fix (`vreader/App/VReaderApp.swift`): `--uitesting` uses `ModelConfiguration(isStoredInMemoryOnly: true)` UNLESS `seedPositionTest || seedKeepExisting` is set. This iteration used `--seed-books` (neither of those), so SwiftData state was in-memory and books were re-seeded fresh on relaunch. **Critically**: `PreferenceStore` writes to `UserDefaults`, which is process-stored to a plist file on the simulator container — independent of the SwiftData decision. Terminate kills the process but does NOT wipe UserDefaults. So the sort preference (`UserDefaults["library.sortOrder"]` or similar key) survived even though SwiftData state did not.
- `--reset-preferences` is the explicit opt-in that DOES wipe UserDefaults via `TestSeeder.clearKnownPreferences(in:)` (added by bug #152 / GH #426). Omitting that flag this round was the whole point — to verify that without explicit reset, prefs persist.
- The pre/post screenshots have identical book-card layout and ordering byte-for-byte modulo the status-bar timestamp; the sort picker checkmark on Date Added (post-relaunch) is the structural confirmation that the active sort is genuinely the persisted value, not just a coincidental ordering.
- The view-mode toggle leg (grid ↔ list) was not driven this round because the iPhone 17 Pro toolbar in iOS 26.4 collapses some `topBarTrailing` items behind a "..." overflow that doesn't directly expose `viewModeToggle` (the `Image(systemName: "list.bullet")` button at `LibraryView.swift:545-555`) on a fresh inspection — only the sort picker shows up under the chevron-right pill. Verifying the toggle would require either CU-driving the iOS overflow specifically (multi-step, finicky) OR exercising it via XCUITest (already covered in round-2's `LibraryPopulatedTests`). Sort-persistence is the substantive persistence claim — view-mode persistence shares the same `PreferenceStore` plumbing per `LibraryViewModel.swift:42` and the surrounding `didSet` hooks at line 105-110, so the persistence guarantee is the same.
- No bugs observed.
