## Observations

- **CU outage handled by the documented fallback.** `/cu-diagnose`
  verdict was virtual-only display ‚Äî the MCP capture path cannot
  capture, but OS-level `screencapture` works. The sim-drive-fallback
  toolkit (single taps + SwiftUI-sheet drags) covered every gesture
  this verification needed; no criterion was blocked by the outage.
- **Simulator contention is real.** The default iPhone 17 Pro
  (iOS 26.5) was being driven by concurrent agents (a
  `vreaderUITests-Runner` was installed there; theme/fontSize state
  flipped mid-run; the vreader app got backgrounded to the home
  screen). Verification was moved to the second iPhone 17 Pro
  (iOS 26.4) for isolation ‚Äî same device class, Dynamic Island, so
  safe-area behavior is identical. iOS 26.4 vs 26.5 is immaterial for
  a SwiftUI sheet re-skin.
- **The planned `Feature61BookDetailsVerificationTests` XCUITest was
  never committed** ‚Äî it is named in the plan ¬ß5 and the GH #800
  comment but does not exist under `vreaderUITests/`. This
  DebugBridge + sim-drive-fallback run covers the same surface (open
  seeded book ‚Üí More ‚ãØ ‚Üí Book details ‚Üí assert sheet + metadata +
  action rows) and additionally exercises the long-title state. Not a
  regression; noted so a follow-up can add the XCUITest if the team
  wants the assertion in CI.
- **`isLongTitle` is computed but unused by any view.** The
  `BookDetailsSheet` header renders `Text(viewModel.title)` with
  `.multilineTextAlignment(.center)` and no `lineLimit` ‚Äî a long
  title wraps natively. The `isLongTitle` flag in
  `BookDetailsViewModel` has no consumer. Harmless (it is
  unit-tested), but it is effectively dead code; a follow-up cleanup
  could drop it. Recorded as a deferred note, not a verification
  failure.
- **Pages row correctly absent for TXT.** Both verified books are TXT
  (reflowable, `totalPageCount` 0/nil) ‚Üí the Pages row is omitted, as
  plan Risk 1 specifies. A PDF book would surface it; not exercised
  here because no PDF fixture was needed for the scoped states.
