# Feature #62 — Annotations panel split — Gate-5 acceptance verification

The unified 4-tab `AnnotationsPanelView` is replaced by two job-focused
sheets routed from the reader bottom chrome: `TOCSheet` (Contents +
Bookmarks navigation) opened by the Contents button, and
`HighlightsSheet` (All / Highlights / Notes / Bookmarks review + a
Share/export button) opened by the Notes button. WI-5 (the final WI)
rewired `ReaderContainerView` to route the two sheets via
`AnnotationsSheetRoute`, deleted the 5 legacy view files, and migrated
the XCUITest consumers.

Verification driven entirely through the XCUITest accessibility API on
the iPhone 17 Pro Simulator — no computer-use. The DebugBridge
`vreader-debug://` scheme was unavailable on this simulator (the
one-shot `lsd` scheme-approval prompt cannot be dismissed headlessly —
`LSApplicationWorkspaceErrorDomain` 115), so the XCUITest suite is the
verification vehicle, which is the appropriate Gate-5 tool for a
UI-flow feature anyway.

## Observations

- **Test-fixture seed bug found and fixed.** The annotations XCUITest
  suites originally used `seed: .books`. `.books` fixtures are
  metadata-only `BookRecord`s with no backing file (`TestSeeder`'s own
  comments cite Bug #209 / #214 / GH #804 / #834) — opening one shows
  "The file could not be found" and the reader bottom chrome (which
  hosts the Contents/Notes buttons) never renders. These suites were
  pre-existing-broken on `main`. WI-5 switched them to `.epubFixture`
  (`mini-epub3.epub` — a real, openable EPUB) with a robust
  `openSeededBook` (3-retry for the `LazyVGrid` layout race) +
  chrome-reveal `app.tap()`, mirroring `Feature63SearchPanelVerificationTests`.
- **Accessibility audit findings.** Running the audit against a real
  `HighlightsSheet`/`TOCSheet` for the first time surfaced: (1) the
  export button's bare 16pt icon had a sub-44pt tap target — fixed
  with `.frame(44,44)` + `.contentShape` (icon visually unchanged);
  (2) `.elementDetection` flags the decorative empty-state SVG art's
  stylized "text-line" bars (`element == nil` — no real control
  missing a label); (3) `.hitRegion` flags `TOCSheet`'s designed
  compact segmented control (mirrors native `UISegmentedControl`,
  also sub-44pt). (2) and (3) are scoped audit exclusions with
  rationale; the export button was genuinely fixed.
- **Out of scope.** `TXTHighlightGestureVerificationTests` and
  `TXTChapterModeHighlightVerificationTests` fail at a pre-existing
  highlight-creation-gesture / chapter-mode-rendering step that WI-5
  never touched (WI-5's delta there is purely the downstream
  identifier swap from the unified panel to `HighlightsSheet`).
  `Feature11EPUBHighlightVerificationTests` self-skips (its
  highlight-creation gesture is XCUI-unreliable — a pre-existing
  documented skip). None is a feature-#62 regression.
