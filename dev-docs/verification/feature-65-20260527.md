# Feature #65 — AI sheet tab-body re-skin — Gate-5 verification (attempt 5, 12/12 PASS)

Feature #65 re-skinned the AI sheet's three tab **bodies** (Summarize / Chat /
Translate) to the v2 design (`dev-docs/designs/vreader-fidelity-v1/project/vreader-panels.jsx`
— `SummaryView` / `ChatView`+`ChatBubble` / `TranslateView`). Implemented +
merged 2026-05-19 (WI-1 #870, WI-2 #871, WI-3 #873, v3.31.0). Four prior
Gate-5b attempts reached **11/12** (then 11.5/12), with the sole residual being
**row 11** — the Translate `.complete` accent-tinted translation result card,
which renders below even the `.large` AI-sheet fold beneath the tall
auto-extracted ORIGINAL card and was un-capturable CU-free.

**This round (attempt 5) closes row 11.** The unblock — a DEBUG-only
`vreader-debug://scroll-sheet?to=<top|bottom>` command (Bug #271 / GH #1188,
shipped in this same v3.39.38) — drives the `TranslationResultCard`
`ScrollViewReader` so the below-fold accent card scrolls into view for
`simctl io screenshot`. With that, all 12 acceptance surfaces now pass →
**VERIFIED**.

## Observations

- Row 11 was the only thing standing between #65 and VERIFIED across four prior
  attempts; each attempt isolated the next missing harness piece (present
  command → AI-action command → detent=large → **scroll command**). The
  `scroll-sheet` command (Bug #271) was the final one — it landed in the same
  v3.39.38 verified here, so this attempt could close the loop immediately.
- `detent=large` alone was confirmed (again) insufficient: the before-scroll
  artifact shows the ORIGINAL card filling the `.large` sheet with the SPANISH
  card only beginning at the very bottom. After `scroll-sheet?to=bottom` the
  full accent card is visible — exactly the gap Bug #271 documented.
- The accent translation card's pixel design matches `TranslateView` precisely
  (accent gradient + accent border + accent semibold target-language label).
  The design's "Speak" button and "Notes on the translation" card are
  intentionally omitted (documented scope reconciliation, not a defect).
- No code defect observed on any surface this round.
