# Feature #67 — Settings profile-header card + Stats entry point (Gate-5b acceptance)

Gate-5b end-to-end acceptance verification for feature #67. The feature
added the design's `ProfileCardLibrary` header card to the top of the
Settings sheet (three-book-spine glyph + "Your library" serif-italic
header + a "N books · Nh read this month" subline + a trailing pill
"Stats" button), restyled the section rows to the design's colored-icon
`SettingsIconRow`, and (WI-6, design #1068 Variant A) restyled the AI
group so the AI Provider + Allow-AI-data-sharing rows are shown only when
the master "Enable AI Assistant" toggle is on.

Verification is **CU-free** — the new suite
`vreaderUITests/Verification/Feature67SettingsProfileCardVerificationTests.swift`
drives the app entirely through the XCUITest accessibility API
(synthesized taps + element queries), which is computer-use-independent.
It does NOT depend on the DebugBridge `present` URL (which cannot be
issued from inside the XCUITest sandbox — Bug #1054). The prior Gate-5a
slice pass (`feature-67-wi6-*-20260521.png`) covered the pixel-level
design fidelity via the sim-drive-fallback screenshot path; this Gate-5b
pass closes the structural + behavioral acceptance criteria as automated
assertions.

## Observations

- The Stats-pill → dashboard assertion is the meaningful cross-feature
  check: it confirms feature #67's entry point posts the hand-off
  notification AND that feature #58's presenter/observer actually mounts
  the dashboard. The `ReadingDashboardView`'s `readingDashboardView`
  identifier did not match the `.otherElements` query but matched the
  `descendants(matching: .any)` fallback — SwiftUI surfaces the
  container as a different element type; the test's `||` fallback handles
  this and passed. (Not a defect — purely a query-type nuance.)
- The AI-group collapse test toggles the real `viewModel.isAIEnabled`
  preference via the `aiToggle` switch — no `--enable-ai` launch arg is
  needed because the master toggle IS the user-facing gate and
  `AISettingsSection` always renders.
- The `.empty` seed renders the subline as "0 books · 0m read this
  month"; the test asserts the stable trailing copy ("read this month")
  rather than a specific count, so it is seed-independent.
