## Observations

- **TXT is the strongest verification.** The war-and-peace fixture's
  chapter 0 is a `"前言"`-class preface (its title is not in the body),
  so it exercises criterion 2 directly — the drop-cap renders on "W"
  with no injected heading and no garbled text. Switching the theme
  Sepia → Dark with the reader open recolored the drop-cap live, which
  proves the `attrStringKey` rebuild seam (criterion 7, TXT).
- **The TXT drop-cap `firstLineHeadIndent` is a touch tight** — the body
  text "ar and Peace" sits close to the oversized "W". This is the
  documented TextKit-1 approximation (plan §3.2 R1 / §8 — no true float);
  it is a faithful *kind* match, not a pixel-perfect float. A future
  tightening of the advance-width estimate is a possible polish item but
  not a defect.
- **EPUB / MD on-device visual not captured this session.** Two named
  tooling gaps: (1) `vreader-debug://open` did not navigate the EPUB
  into the reader on the shared simulator (the reader stayed on Library;
  `settle` reported `no active reader`) — the EPUB reader's open path
  behaves differently from TXT's under the bridge; (2) there is no MD
  fixture in `DebugFixtureCatalog`, and the Inbox-`openurl` import path
  did not trigger vreader's import pipeline. `mcp__computer-use__screenshot`
  returned `CU display unavailable` throughout, so the EPUB cover could
  not be tapped manually. The EPUB drop-cap is a pure CSS-string addition
  (10 tests pin selector + declarations + per-theme color) riding the
  already-shipped EPUB injection pipe; the MD decorator is covered by 43
  tests. These are the `partial` criteria.
- **AZW3 confirmed-injecting.** `mini-azw3.azw3` (*The Masque of the Red
  Death*) opened in the Foliate reader with the drop-cap rule injected —
  no crash, correct rendering. The fixture's first `<p>` is section-
  wrapped (Project Gutenberg markup), so `body > p:first-of-type::
  first-letter` is a documented safe-miss (no drop-cap rather than a
  wrong one — plan §4.5 / R6). A flat-top-level-`<p>` AZW3 would get the
  drop-cap.
- **Concurrent-PR version collisions.** During this run `main` advanced
  rapidly; WI-1/WI-2/WI-3 each merged sharing a version line with a
  concurrent PR (the `vX.Y.Z` tag for those lines was already taken).
  WI-4/WI-5 bumped to unique versions (3.31.12 / 3.31.13). The feature
  code is unaffected — this is a tagging artifact of parallel merges.
