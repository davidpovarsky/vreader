## Observations

- **The decisive finding** is eval-proven, not a screenshot guess: the drop-cap
  CSS rule is present but its `body > p:first-of-type` selector is structurally
  incompatible with the #71 continuous-stitch DOM (per-spine DIV wrappers). Since
  continuous scroll is a primary EPUB mode, users do not see the drop-cap the
  feature promises there. This was missed by the prior round because it verified
  in a paged/single-section render where `<p>` is a body child.
- MD's renderer (markdown-native UITextView) restyles correctly in scroll mode;
  the EPUB defect is specific to the WebView-CSS-selector path under continuous
  windowing.
