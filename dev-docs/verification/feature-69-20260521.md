## Summary

Gate-5b verification of feature #69 — **attempt 2**, re-running on
v3.38.29 after Bug #237 (`--enable-ai` consumed via
`AITestOverride.forceAvailable`) AND Bug #243 (DebugBridge `provider`
URL family) both shipped. Goal: unblock criteria 7 / 8 from
`feature-69-20260519.md` (the end-to-end on-device acceptance — open
AI sheet, tap a scope chip, observe scoped summary render).

**Result: `partial` (unchanged from prior round).** Same blocker
shape as feature #65 (`feature-65-20260521.md`) — the provider URL
ships and `--enable-ai` reaches the AI surfaces, but a host-driven
provider configuration cannot be interleaved with an XCUITest's
launch-to-action sequence inside the same process (Bug #1054). The
end-to-end scope-chip → AI-response chain remains structurally blocked
on this virtual-display-only host until either CU returns or a hybrid
host-driver / XCUITest harness is built.

Criteria 1-6 (the implementation layer) all PASS by the 163-test
deterministic unit suite, unchanged from 2026-05-19. The
chip-strip rendering / tap-selection / no-auto-fire / `runSummarize`
forwarding / a11y IDs are all covered by `AISummaryTabViewScopeTests`
+ `AISummaryTabViewTests`. The only criteria that need end-to-end UI
are 7-8, and those need the provider response to render.

## Observations

- **CU is structurally down for the 3rd consecutive verify-cron
  session.** See `feature-65-20260521.md` § Observations for the
  full diagnosis. Same condition — `sim-drive-fallback` CGEventPost
  taps do not translate; XCUITest does.
- **The 163-test deterministic logic suite stays PASS on v3.38.29.**
  No code-level changes to feature #69 since 2026-05-19; the same
  passing evidence carries forward.
- **The wall is now precisely named.** Criteria 7-8 need
  end-to-end: (a) app launched with `--enable-ai`, (b) provider
  configured via host-side `vreader-debug://provider?action=add`,
  (c) XCUITest opens the reader's AI sheet, taps a scope chip,
  observes the resulting summary card. (a) and (c) live inside an
  XCUITest method; (b) MUST run from a host shell (Bug #1054). No
  current harness interleaves them.
