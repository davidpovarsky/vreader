# Feature #72 — HTTP cloud TTS provider integration

Wires the previously-orphaned `HTTPTTSProvider` (Bug #270 / GH #1166) into the
live read-aloud path so a configured cloud-TTS endpoint actually speaks. Also
corrects Feature #26's C6 gating (C6 was unimplemented, not merely unverifiable).

## Verification approach — close-gate exception

Acceptance criterion C6 ("end-to-end verified") would normally require a live
third-party HTTP TTS server. None is available (external infrastructure). Per the
AGENTS.md close-gate **verification-exception** path, this feature is verified by
a deterministic high-fidelity integration test that drives the SAME code paths a
real cloud read-aloud would hit — `TTSService → HTTPSpeechSynthesizer →
HTTPTTSProvider` through their real subsystem boundaries — stubbing only the
network **transport** (`URLSessionProtocol`) and the audio **hardware**
(`SpeechAudioPlaying`). The test records and asserts the real per-chunk HTTP
request contract (method, URL, header, expanded body), so it is not a wiring-only
stub test.

Test file: `vreaderTests/Integration/Feature72CloudTTSIntegrationTests.swift`.

## Observations

- The verification gate did its job: the failure-path integration test (C5)
  exposed a genuine end-to-end defect that WI-3's adapter-level unit tests
  missed. The adapter correctly fired `didCancel` on failure, but `TTSService`'s
  delegate handler swallowed it because the `state == .speaking` heuristic could
  not distinguish a restart-cancel (outgoing utterance) from a terminal
  failure-cancel (current utterance). This is exactly the class of bug
  integration tests exist to catch.
- Codex Gate-4 round-2 caught a subtle leak in the first fix attempt: arming the
  counter from `state` (which lags the backend through a Task hop) would leak
  when a restart followed natural completion. Re-armed from the `stopSpeaking()`
  Bool result instead. The `MockSpeechSynthesizer` had to be corrected to return
  `isSpeaking || isPaused` (it returned `true` unconditionally) to faithfully
  model the real synthesizers' contract.
- `TTSControlBar` pause/resume/stop is shared across all readers and was already
  device-verified end-to-end in Feature #26 round-6; the cloud path reuses it
  unchanged, so C2's control-bar slice did not need re-driving on a device.
