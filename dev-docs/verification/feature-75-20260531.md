# Feature #75 — RTL/vertical EPUB paged rendering — device verification

Slice verification of the paged-rendering of **both** the horizontal-RTL and
vertical-rl axes on device, driven CU-free via `scripts/sim-tap.sh` +
`xcrun simctl io` screenshots. Page-turn input for the vertical axis is the
remaining WI-5 work (see Observations).

## Observations

- The horizontal-RTL path (WI-1 axis resolution → WI-2 `direction: rtl` CSS +
  negative-scrollLeft → WI-3 pre-pagination probe → WI-4 tap-zone mirror) works
  end-to-end in production: RTL content renders right-aligned in paged columns,
  and a left-zone tap advances (RTL reading order).
- LTR regression check passed — the probe (now on every paged setup) did not
  break normal EPUB reading.
- **Vertical-rl RENDERS correctly** (verified after bypassing the harness
  friction with a clean `simctl uninstall`+`install`+seed): `mini-cjk` chapter 2
  shows Chinese text in vertical columns, top-to-bottom, right-to-left, in paged
  mode. The per-document probe correctly resolves ch1 horizontal vs ch2 vertical
  in the same book. So WI-2's `writing-mode: vertical-rl` CSS + WI-3's probe are
  confirmed on device.
- **Vertical page-turn DEVICE FINDING (open)**: authored a new multi-page
  fixture `mini-cjk-vlong` (one long `writing-mode: vertical-rl` chapter, 24 CJK
  paragraphs) to unblock this. On-device paging through it revealed vertical-rl
  paged navigation is INCORRECT under the current shared negative-`scrollLeft`
  sign: the initial open showed later paragraphs (20-24), and a left-zone tap
  jumped to the chapter heading + paragraphs 1-2 — i.e. the initial position +
  turn direction are inverted. `writing-mode: vertical-rl` column pagination does
  NOT follow horizontal-RTL's WebKit scroll convention. The correct sign / tap /
  swipe coupling needs iterative on-device tuning against `mini-cjk-vlong`;
  pinned in `EPUBPagedAxisTests.verticalRL_currentSignMatchesRTL` so the fix is a
  deliberate, tested change. RENDERING of both axes is verified; this is the
  page-turn INPUT gap.
- **Harness gap (file as DevTools/Verification)**: `vreader-debug://reset` does
  not clear the imported-books library, so seeding a second fixture leaves the
  first; a clean `simctl uninstall`+`install` is the current workaround.

## Correction (eval-grounded)

An earlier revision of this file claimed vertical-rl RENDERS correctly. On-device
`eval` (`getComputedStyle`) disproves it: writing-mode is `horizontal-tb`
throughout, so vertical-rl is NOT actually rendering — the visual was narrow
multi-column horizontal CJK. The real #75 vertical gap is twofold and deeper than
a pagination sign: (1) the EPUB reader must PRESERVE + apply the book's
`writing-mode` (head `<style>` is currently dropped when content is extracted into
the wrapper div), and (2) `navigateToPageJS` must target the actually-scrollable
element (the content div has horizontal overflow `sw:2052`; the top document does
not). Only the horizontal-RTL axis is genuinely verified.
