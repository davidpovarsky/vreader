# Feature #77 ‚Äî Gate-5b: inline bilingual loading shimmer (EPUB/AZW3)

`status_target: DONE` ‚Äî this run does NOT flip the row to `VERIFIED`. Two of
three engine families fully pass (Readium default + legacy EPUB paged+continuous);
the Foliate (AZW3) translation-replace half could not be confirmed CU-free with
the current single-line mock. Row stays `DONE` with the Foliate slice deferred.

## Method

Verified through the new `bilingual?action=enable|disable|status` DebugBridge
harness shipped in this PR (v3.59.30), CU-free. `--mock-ai
--mock-ai-translate-delay-ms=<N>` holds the translate in-flight so the loading
shimmer is snapshottable before it lands. Per engine: enable bilingual ‚Üí
snapshot `inFlight` (the shimmer's source signal) + eval the live DOM for
`.vreader-bilingual-loading` (shimmer) vs `MOCKËØë` (landed translation), during
and after the in-flight window.

Real-books-first note: the seedable bundled `mini-cjk` (CJK EPUB) and `mini-azw3`
(AZW3) fixtures were used ‚Äî they satisfy "CJK book per engine" and are the only
CU-free-seedable path (a real-import has no CU-free fingerprint-key discovery
seam). Engine routing confirmed via the harness's `engine` readout field.

## Observations

- **Engine routing**: EPUB default layout is **scroll** ‚Üí legacy continuous host
  (`engine: epub-legacy`); paged layout ‚Üí Readium (`engine: readium`, the default
  reflowable engine). Both `readiumEPUBEngine` + `epubContinuousScroll` flags
  default ON. The `--reader-default-layout=paged` pre-seed only applies under
  `--uitesting` (gated at VReaderApp.swift:217).
- **The shimmer renders on all three engine families** ‚Äî confirmed by DOM
  `.vreader-bilingual-loading` counts (3 / 6 / 9) and, for Foliate, a visual
  screenshot (gray interlinear shimmer bars under each block, `EN‚ÜîIt` toggle
  active).
- **Foliate translation-replace gap**: `inFlight` cycled 0‚Üí2‚Üí0 (prefetch units
  started + finished) but the foliate-frame DOM stayed `loading:9, mock:0` even
  after >37s ‚Äî the translation never injected to replace the shimmer. Root cause
  is most likely a **mock-harness limitation, not a product defect**:
  `ChapterTranslationService` requests a JSON-array of N segments per chunk and
  on a decode failure falls back to one-request-per-segment; `MockAIProvider`
  returns a single non-JSON line. On the EPUB engines the units were
  single-paragraph (the per-segment fallback matched 1:1 ‚Üí `mock:6`/`mock:3`),
  but Foliate's units are multi-paragraph (9 blocks across 2 in-flight units), so
  the single-line mock can't satisfy the multi-segment expectation within the
  observed window. Confirming this (vs a real Foliate inject defect) needs a
  multi-segment mock ‚Äî filed as a follow-up.
