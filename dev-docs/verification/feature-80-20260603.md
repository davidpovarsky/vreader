Device verification on iPhone 17 Pro Simulator (UDID
`61149F0E-DC18-4BE2-BB37-52659F1F4F62`) of the merged build on `main`
(`ce4543fc`, v3.49.0 build 831). Path: Library → Settings → AI Providers → **+**
(Add Provider) → scroll to API Key + Connection.

## Acceptance criteria

| # | Criterion | Observed | Result |
|---|---|---|---|
| 1 | Add-mode with an API key typed → Test enabled, tests live form + typed key | The **Test Connection button now renders in ADD-MODE** (was edit-mode-only). | 3 | States: idle → Testing… (spinner) → Connected (green) / Failed (red) | The spinner + green/red result rendering is present in `testConnectionSection`; driving Connected/Failed needs a real key + network (prohibited key entry). The `testConnection` success/failure mapping + the in-flight `testGeneration` guard are unit-verified. | PASS (unit) |
| 4 | Edit-mode with a saved key (no typed key) still tests via keychain (no regression) | UNIT: `nilOverride_fallsBackToKeychain` (Authorization == the keychain key when override nil). | PASS (unit) |
