## Observations

- The bilingual setup sheet only shows on FIRST enable per book; per-book bilingual
  config persists by deterministic fixture fingerprint across `reset`, so reaching the
  first-enable sheet needs a not-yet-configured fixture (used `mini-azw3` / `war-and-peace`).
- `aiAssistant` + consent are PERSISTED flags; a prior in-sheet AI-toggle-off persists, so
  the gate state on a later launch reflects that ‚Äî handled correctly (the tracker showed the
  true live state).
- The AI panel opening with the active provider name is the load-bearing proof of the
  `AIReaderAvailability` per-profile alignment (the #308 loop fix).
