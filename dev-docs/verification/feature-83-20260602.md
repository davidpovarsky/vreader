# Feature #83 — Readium cross-chapter continuous scroll — Gate-5 acceptance

Device verification on iPhone 17 Pro Simulator (UDID
`61149F0E-DC18-4BE2-BB37-52659F1F4F62`), v3.45.0 build 816 (== merge `6c3d1efa`).
Readium EPUB engine (`readiumEPUBEngine` ON), scroll layout
(`--reader-default-layout=scroll`), `multi-chapter-epub` fixture. Driven via idb
`scripts/sim-tap.sh` swipes; the boundary observer / handler logged via OSLog.

Resolves **Bug #309 (#1403)** — the regression of VERIFIED Feature #71 from the
Readium default flip.

## Observations

- ALPHA / BRAVO / CHARLIE are long synthetic chapters (~20% of the book each), so
  reaching a boundary takes many swipes — but each boundary crossing was a single
  clean auto-advance.
- The Gate-2 honest framing held: there's a visible resource-transition seam at the
  boundary (Readium's `slideToView`), not pixel-continuous stitching — but the
  "stops at the boundary, must swipe" friction (#309) is gone.
- The in-flight guard (cleared on the nav-settle + `locationDidChange`) correctly
  allowed exactly one advance per boundary across multiple chapters.
