# Feature #84 — Secondary-text `sub`-token AA bump (Paper/Sepia) — Gate-5 acceptance

Verification of the merged build on `main` (merge commit `482be9f0`, v3.46.0
build 823). Implements the landed design
`dev-docs/designs/vreader-fidelity-v1/project/design-notes/secondary-text-sub-token.md`
(resolves the closed needs-design #1292, the secondary-text facet of Bug #285).

This is a theme-token / WCAG-contrast change: the acceptance criteria are
contrast ratios, which are deterministic and CU-free verifiable through the
contrast unit suites — the same quantitative-contrast verification path used for
the sibling Display-panel facets #1273 (`sliderTrack`) and #298 (`controlTrack`).

## Observations

- The fix is a strict improvement on every surface (alpha-up raises contrast
  everywhere `sub` renders), not just the Display sheet.
- Gate-4 Codex audit (`019e88ef`) caught a missed `a:visited` test pin (fixed)
  and confirmed no other runtime/test pin of light-family `sub`=0.55 remained.

## Known follow-up (surfaced to the user, not filed)

Per the design note's own out-of-scope section + the Gate-4 audit: Sepia
`chromeColor` (~4.27:1) / `paperColor` (~4.39:1) `sub` text — improved from
~3.36:1 but still short of AA on those non-sheet surfaces — and Dark/OLED
secondary (~3.06–3.75:1 over `#222020`) are separate visual-weight design
decisions, not this slice's scope. The unified `sub` token cannot clear AA on
every surface without either a darker value (crowds primary) or per-surface
tokens (the alternative the design explicitly rejected). Recommended as a matched
follow-up; surfaced to the user rather than filed unilaterally.
