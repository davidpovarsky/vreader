# Feature #86 — Expand AI reading scope & sources (Chat tab)

Clears the one residual that held this feature at `DONE`: "the whole-book read
*answering* + citation *content* are provider-key-blocked → keyed-verification
before VERIFIED." The DEBUG `MockAIProvider` (`--mock-ai`, v3.59.10, PR #1554)
injects a deterministic, **key-free** provider into `AIService`, so the in-reader
AI Chat actually answers — driving the real reader → AIService → provider →
streamed-reply → UI-render path **without a real API key** (which an automated /
headless run is prohibited from entering). The end-to-end answering is now
exercised by a CU-free XCUITest.

## Observations

- The reader AI panel's container `.accessibilityIdentifier("aiReaderPanel")`
  propagates onto descendants (Bug #209/#214), shadowing the chat element IDs —
  so the test queries by element TYPE + accessibility LABEL (textView composer,
  button label "Send", staticText CONTAINS "[MOCK]"), which the container does
  not shadow. The reader chrome also auto-hides, so the AI button is revealed
  with a center tap + hittability retry before opening the panel.
- The `[MOCK]` reply carries a context-length reflection, proving the answer
  came through the real `AIService.streamRequest` → resolved-provider path (the
  mock is injected at `resolveProvider`), not a UI stub.

## Verification approach (transparent)

The answering residual is verified with the **key-free** `MockAIProvider`, not a
real provider key. This is a deliberate, documented choice: entering an API key
is a prohibited action for an automated run (AGENTS.md), and the assertable
acceptance — *the AI produces an answer from the assembled scope and it renders*
— is fully exercised by the mock through the real subsystem boundaries. The
non-deterministic quality of a real LLM's prose is NOT an acceptance criterion;
the real provider HTTP/SSE wire format is separately covered by
`OpenAICompatibleProvider` / `AnthropicProvider` unit tests. The mock is the same
class of deterministic verification affordance as the #74 bloom harness.
