# Feature #89 — Back up AI conversation sessions & history via WebDAV — Gate-5b acceptance

Additive `ai-conversations.json` WebDAV backup section that round-trips the #88
`ChatSession` rows (byte-exact message blob) through the existing collect→ZIP→restore
flow. No UI, no new `@Model`. Verified CU-free by re-running the shipped Swift Testing
suites on merged main — they exercise every acceptance criterion through the REAL
subsystem boundaries (collector, restorer, PersistenceActor actor methods, the
WebDAVProvider orchestration loop) against an in-memory container + in-memory provider,
which is the appropriate end-to-end substitute for this no-UI, additive backend section.

## Observations

The 18 `BackupAIConversationsTests` + 35 `WebDAVProviderTests` drive the production
`PersistenceActor+ChatSessionsBackup` upsert through a real in-memory `ModelContainer`
on `Schema(SchemaV9.models)` — the same SwiftData boundary production hits — so the
re-key / book-release / never-clobber / idempotency behaviors are exercised against the
real persistence layer, not stubs. The non-UTF8 byte-exactness test specifically guards
the Gate-2 `Data?` (vs lossy `String?`) decision. No live WebDAV server is needed because
the section is additive metadata JSON inside the existing ZIP; the orchestration tests
confirm it is actually wired into `backup`/`restore`, not just the collector in isolation.
