- The context menu's destructive coloring (Remove Cover + Delete in red) is visually consistent with iOS Human Interface Guidelines for destructive actions inside contextual menus.
- "Add to Collection" surfaces the chevron-right disclosure indicator without being tapped this round; tap-to-expand-submenu was not driven this iteration since collections sidebar UI verification was completed separately under feature #34's round-2 (commit `e738ae6`, PR #425).

"Remove Cover" tap-flow not driven this iteration; cosmetic menu surface only — implementation reads from `CustomCoverStore`, well-covered by unit tests.
- Share sheet's file label `epub_0000000000000000…` reflects the fingerprint-keyed filename produced by the import flow. The seeded fixture has a real file on disk under `ImportedBooks/` (the `--seed-books` flow inserts the book record AND lays down a placeholder bytes file at the canonical-key path so `LibraryBookItem.resolvedFileURL` resolves), so Share is correctly gated on `.local` rows per feature #47 WI-5.
- No bugs observed. No regressions vs round-2.
- iOS shows the Dynamic Island time as `01:42` / `01:43` during the capture window — confirms the screenshots are timeline-consecutive rather than cached.
