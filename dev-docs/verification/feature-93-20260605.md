# Feature #93 — AZW3/MOBI (Foliate) theme-color parity — VERIFICATION

Device verification of the AZW3/MOBI Foliate theme-color parity shipped in
v3.53.0 (WI-1, PR #1504, merge `52c405a2`). Verified against the **exact book
the user reported** — 被讨厌的勇气 (`Bei Tao Yan De Yong Qi - Zi Wo.azw3`),
imported **native** (Foliate-rendered) by disabling `kindleConvertOnImport` so
it does NOT convert to EPUB→Readium (the #93 gap is exactly "existing native
`.azw3` books that still render via Foliate").

## Acceptance criteria

| Criterion | Observed | Pass |
|---|---|---|
| The theme paper/sepia/dark background replaces the publisher's embedded `background-image` (the blue watercolor) | Dark theme → reader renders on a dark background (no blue watercolor); light/paper theme → reader renders on a paper/cream background. The publisher background does not show in either appearance. | ✅ |
| Body text uses the theme ink (legible on dark) | Dark theme → light/cream ink on dark (legible); paper theme → dark ink on paper. | ✅ |
| Live theme switch re-tints without reopening the book | Switching dark → light via `vreader-debug://theme` re-tinted the open Foliate reader (background paper, ink dark) with no reopen — the `updateUIView` `setStyles` diff path. | ✅ |
| Native AZW3 routes through Foliate (not converted) | AX tree reported the book as "AZW3 format" after importing with `kindleConvertOnImport` OFF (the default-ON convert path would have shown "EPUB format"). | ✅ |

## Commands run

```bash
SIM=61149F0E-DC18-4BE2-BB37-52659F1F4F62
SIMCTL=/Applications/Xcode.app/Contents/Developer/usr/bin/simctl

# Install the merged-main DEBUG build (resolved from BUILT_PRODUCTS_DIR)
$SIMCTL install "$SIM" "<DerivedData>/Debug-iphonesimulator/vreader.app"

# Relaunch with convert-on-import OFF so the AZW3 stays native (Foliate path)
$SIMCTL terminate "$SIM" com.vreader.app
$SIMCTL launch "$SIM" com.vreader.app -com.vreader.featureFlags.kindleConvertOnImport NO
$SIMCTL openurl "$SIM" "vreader-debug://reset"

# Import the real AZW3 via the app Inbox
APP_DATA=$($SIMCTL get_app_container "$SIM" com.vreader.app data)
cp "test-books/books/azw3/Bei Tao Yan De Yong Qi - Zi Wo.azw3" "$APP_DATA/Documents/Inbox/beitaoyan.azw3"
$SIMCTL openurl "$SIM" "file://$APP_DATA/Documents/Inbox/beitaoyan.azw3"
# AX tree confirmed "AZW3 format" (native, not converted)

# Dark theme + open the book + screenshot
$SIMCTL openurl "$SIM" "vreader-debug://theme?mode=dark&fontSize=22"
bash scripts/sim-tap.sh xy 77 400        # tap the book cell
$SIMCTL io "$SIM" screenshot /tmp/verify-93-azw3-dark.png

# Live-switch to paper theme + screenshot
$SIMCTL openurl "$SIM" "vreader-debug://theme?mode=light&fontSize=22"
$SIMCTL io "$SIM" screenshot /tmp/verify-93-azw3-light.png
```

## Observations

- The convert-on-import default (#42) means a *freshly imported* AZW3 becomes
  EPUB→Readium and would NOT exercise #93 — the first import showed "EPUB
  format". Disabling `kindleConvertOnImport` via the NSArgumentDomain launch
  arg (`-com.vreader.featureFlags.kindleConvertOnImport NO`) was required to
  reproduce the native-Foliate path #93 targets. Documented so the next
  verifier doesn't trip on it.
- The copyright (版权信息) first content page rendered cleanly themed in both
  appearances; the background tracked the theme, not the publisher.
- Live theme switch worked on the already-open reader — the `setStyles` diff
  in `updateUIView` re-applied the new theme colors without a reopen.
