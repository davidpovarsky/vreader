# Feature #94 — Filterable TOC (Contents-tab filter field) — Gate-5b acceptance

Client-side filter field pinned atop the Contents tab of `TOCSheet`: narrows the
loaded chapter entries by title as you type (case-insensitive, diacritic-folded,
CJK-aware substring), with a live "N of M chapters" count, match highlighting, a
current-chapter row pin, a clear-restore re-scroll, and a no-match "Open Search"
escape hatch. The behavior is pure client-side filter logic plus host styling — no
new `@Model`, no persistence, no network. Verified CU-free by re-running the shipped
Swift Testing suites on merged main (v3.59.0); they exercise every acceptance
criterion through the real `TOCTitleFilter` filter engine and the real `TOCSheet`
view-model boundaries.

## Observations

Feature #94 is pure client-side filter logic — there is no persistence, network, or
new `@Model`, so the unit/view suites ARE the appropriate end-to-end substitute:
`TOCTitleFilterTests` drives the production `TOCTitleFilter` engine directly
(case-insensitive / diacritic-folded / CJK single-character substring matching, the
"N of M chapters" / "No chapters match" count label, the match-range computation that
returns ALL occurrences mapped back to the ORIGINAL un-folded title, the current-row
pin that carries the ORIGINAL `tocEntries` index under filtering, and the
clear-restore predicate that re-fires the current-row auto-scroll only on the
transition back to an empty query). The Gate-2 audit's two subtle pre-implementation
bugs are both guarded here: `filteredPreservesOriginalIndices` proves a filtered row
does NOT renumber (so a tap navigates to the right chapter), and
`diacriticRangeMapsToOriginal` proves the highlight ranges land on the original
accented run rather than the NFKC-folded form. `TOCSheetTests` covers the host side —
current-chapter accent+bold styling across every theme, the Bookmarks-tab isolation
(badge + empty state independent of the Contents filter), and the Open-Search CTA the
no-match escape hatch reuses. Everything is deterministic and CU-free; no synthetic
fixture beyond the in-test `TOCEntry` arrays is needed because the filter operates on
already-loaded entries.
