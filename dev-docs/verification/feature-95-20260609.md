# Feature #95 — Justified text alignment for the EPUB reader

## Acceptance criteria

| # | Criterion | Observed | Pass |
|---|---|---|---|
| 1 | Readium EPUB (paged, default): body paragraphs justified | CJK `<p>` computes `text-align: justify`; screenshot shows flush right margins | ✅ |
| 2 | Legacy EPUB (continuous #71 stitch): body paragraphs justified | CJK `<p>` in the stitch computes `text-align: justify` | ✅ |
| 3 | AZW3/MOBI Foliate: body paragraphs justified | Latin prose renders with flush right margins (screenshot) | ✅ |
| 4 | CJK EPUB: both margins flush (motivating case) | mini-cjk first paragraph flush both margins on Readium + legacy | ✅ |
| 5 | Headings NOT force-justified | "第一章" heading stays centered while paragraphs justify | ✅ |

## Commands run

```bash
SIM=61149F0E-DC18-4BE2-BB37-52659F1F4F62
# Legacy continuous (default scroll):
xcrun simctl openurl "$SIM" "vreader-debug://seed?fixture=mini-cjk"
xcrun simctl openurl "$SIM" "vreader-debug://open?bookId=<epub-key>"
xcrun simctl openurl "$SIM" "vreader-debug://eval?bridge=epub&js=<getComputedStyle(p).textAlign>"
# → {"textAlign":"justify","sample":"这是一个中文测试电子书的第一段。…"}
# Readium paged:
xcrun simctl launch "$SIM" com.vreader.app --uitesting --uitesting-no-seed --reader-default-layout=paged
# → {"textAlign":"justify","sample":"这是一个中文测试电子书的第一段。"}
# AZW3 Foliate: seed mini-azw3, open, seek 0.5, screenshot → prose flush
```

## Observations

- Both EPUB engines (Readium + legacy) measured conclusively via `getComputedStyle(p).textAlign == "justify"`.
- Foliate's content-frame `getComputedStyle` is unreliable (per the #68 finding — `setStyles` isn't reflected there), so AZW3 was verified VISUALLY (the screenshot shows flush right margins on the Latin prose). The rule ships in the same `themeCSS` batch as the visibly-applied font/color rules.
- The "第一章" heading stays centered, confirming the `p`-only selector leaves headings alone.
