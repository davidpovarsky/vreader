## Observations

- **Device-verification finding (fixed before merge):** the original export trigger
  embedded a `UIActivityViewController` (`ShareActivityView`) in a `.sheet`. From
  this presentation depth (Settings `.sheet` â†’ `NavigationStack` push â†’
  `.sheet`), it rendered a **blank white sheet** â€” `UIActivityViewController`
  does not present correctly when embedded three levels deep (the shipped
  `HighlightsSheet` works only because it's one sheet level deep). Refactored to
  the native iOS-17 `ShareLink` with a reactively-prepared redacted file URL
  (regenerated off-main on appear + each filter change). Re-verified: the system
  share sheet now presents correctly with the `.txt` payload. This is exactly the
  class of bug device verification exists to catch.
- **Simulator OSLogStore caveat:** `OSLogStore(scope: .currentProcessIdentifier)`
  returns ZERO entries until the app emits a persisted-level (`.notice`+) `Logger`
  line this session; `.info`/`.debug` are not persisted by default. Opening a book
  produced real `.error` (translation/prefetch TLS) + persisted entries, which the
  viewer then displayed â€” confirming the live read-back path works on the sim.
- Functional level colors render correctly (INFO blue, ERROR red); the messages
  visible (TLS errors, FoliatePosition CFI saves) contain no secrets, and the
  export redaction is independently unit-tested (`DiagnosticsRedactorTests`).
