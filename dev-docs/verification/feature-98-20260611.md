## Observations

- **The simulator does not fire background-task expiration handlers** and
  does not honor the ~30s grace duration (suspended ~5s into background
  with tokens held, twice). It DOES suspend backgrounded processes — a
  mid-request suspension produced the exact production failure from the
  user report (request timed out on resume → `.failed`/PAUSED), which is
  how the PAUSED rendering got device-verified. Real-device grace/expiry
  semantics are Apple-implemented; our side of the contract (begin/end
  pairing, handler → latch → between-units stop → descriptor) is pinned
  by the deterministic suites.
- **cfprefsd gotcha** (matches the standing memory on defaults domains):
  editing the app plist while the sim is booted gets clobbered by the
  preferences daemon's cached view at next app launch — the descriptor
  seed only sticks if written while the simulator is SHUT DOWN.
- The resume skipped ~370 cached units in ~1s (banner first observed at
  1037/1860 includes cache-skip ticks) — `cachedUnits` as the checkpoint
  works exactly as planned; no re-translation of completed units.
