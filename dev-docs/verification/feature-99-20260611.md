# Feature #99 — Translation settings re-entry — Gate 5b acceptance

Plan: `dev-docs/plans/20260611-feature-99-translation-settings-reentry.md`.
Design: `vreader-bilingual-suite.jsx` (`BSMorePopover` / `BSSettingsSheet` /
`BSLangTile` / `BSCostStrip` / `BSRetranslateBanner` / `BSBilingualPill`),
bundle PR #1652, design-notes §#1640. All 4 WIs merged: WI-1 PR #1673
(v3.65.3), WI-2 PR #1674 (v3.65.4), WI-3 PR #1675 (v3.65.5), WI-4 PR #1678

## Observations

- The provider segment resolved to the live active profile ("Mock") on
  every popover open — the generation-stamped fetch never flashed stale
  copy.
- The pill flip is immediate on apply (the `.readerBilingualDidChange`
  mirror), and the MOCK-translated rows from the previous language stay
  on screen until the prefetch re-lands — exactly the designed
  "re-translate as you read" semantics.
- 160 feature-#99 unit tests green across the 4 WIs (edit-model matrix,
  store cachedLanguages, sheet composition pins incl. first-enable
  byte-identity, row/notification/effect contracts, router
  dirty-before-apply + banner payload, fetcher races).
