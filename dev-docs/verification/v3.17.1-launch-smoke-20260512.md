## A near-miss false-positive worth noting

Initial dark-theme probe returned LIGHT theme CSS (`bgColor: rgb(255,255,255)`, `color: rgb(25,25,25)`) even after `theme?mode=dark` was persisted. Looked like a fresh bug. Investigation traced it to a **leftover per-book settings file** at `Library/Application Support/PerBookSettings/epub_f284fd...:2198.json` pinning `themeName: "light"` for the test fixture. After removing the file, the renderer correctly used the global dark theme.

This is NOT a bug — per-book overrides are designed to outlive DebugBridge `reset` (which wipes the SwiftData library but not the per-book settings sidecar files, since the user might re-import the same fingerprint). Future cron-driven verifications that toggle global theme via DebugBridge should wipe `Library/Application Support/PerBookSettings/` first if they need to observe global-theme behavior without per-book interference.

A small follow-up improvement could be to extend `vreader-debug://reset` to also wipe per-book settings sidecar files — but that's a feature/enhancement request, not a bug. Not filed this iteration (verification scope).

# After: clean per-book sidecar (was pinning to light from prior session)
DATA=$(xcrun simctl get_app_container $SIM_ID com.vreader.app data)
rm "$DATA/Library/Application Support/PerBookSettings/epub_f284fd...:2198.json"

# Open + probe
BOOK_KEY="epub:f284fd074ccd1d3c1a78985464d9e1be27975f4029f3c2ddef8428ca10684af4:2198"
ENCODED=$(printf '%s' "$BOOK_KEY" | sed 's/:/%3A/g')
xcrun simctl openurl $SIM_ID "vreader-debug://open?bookId=$ENCODED"
xcrun simctl openurl $SIM_ID "vreader-debug://settle?token=open"

JS='var b=document.body; var s=getComputedStyle(b); JSON.stringify({paddingTop: s.paddingTop, paddingBottom: s.paddingBottom, bgColor: s.backgroundColor, color: s.color, firstElTop: b.firstElementChild?.getBoundingClientRect().top, columnCount: s.columnCount, bodyHeight: b.clientHeight});'
B64=$(printf '%s' "$JS" | base64)
xcrun simctl openurl $SIM_ID "vreader-debug://eval?bridge=epub&js=$B64"
cat "$DATA/Library/Caches/DebugBridge/eval-epub.json"
