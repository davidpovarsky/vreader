# Architecture Decision Records (ADRs)

Durable, forward-looking decisions a future agent or contributor reads to
understand *why* the project is shaped the way it is — distinct from the live
trackers (`docs/bugs.md`, `docs/features.md`) and from `docs/architecture.md`
(which describes the *current* codebase, not future direction).

One file per decision, `NNNN-<slug>.md`, append-only (supersede with a new ADR
rather than rewriting history).
